# 🏆 EduMatrix 智教矩阵 — 双轨追问交互与独立自由悬浮舱方案 (最终确认版)

根据您的最终批准，系统将实现以下双轨追问架构：

1.  **卡片整块追问 (卡片级) ➡️ 豆包式主输入框引用**：
    *   点击卡片底部的 `💬 追问` -> 载入主输入框上方的引用预览条，在主对话流里进行多轮追问与回答。
2.  **选区划词追问 (划词选区) ➡️ 独立悬浮追问舱**：
    *   划选文字后点击浮动小胶囊徽章 -> 弹出一个**独立的、无遮罩遮挡、支持自由拖拽位置的悬浮追问舱窗口**。
    *   **上下文静默等待**：打开追问舱后，窗口顶部仅以静态形式展示划选文本作为上下文，**不触发自动 AI 回答**。
    *   **舱内闭环对话**：用户在舱体输入框中输入问题，发送后智能体的流式响应直接在**追问舱内部渲染**，保持划词追问交互完全在悬浮舱内闭环。

---

## 一、 后端接口兼容性说明

我们将继续调用已有的异步流式接口 `/api/stream/explain`。
该接口接收：
- `target_text`: 选中文本
- `context_before` / `context_after`: 上下文
- `follow_up`: 追问问题
- `history`: 舱内历史对话记录

舱内的多轮对话会持续拼装历史上下文，并通过 SSE 流式推送到追问舱内部。

---

## 二、 前端重构设计与改动

### 1. 恢复与挂载 `InlineSocraticPopup.vue`
- 我们需要恢复 `InlineSocraticPopup` 组件，并将其挂载在 `Chat.vue` 的模板底部。
- **划词胶囊点击事件**：
  划词胶囊徽章点击时，重新触发 `openSocraticPopup(textSelection.text)`。
  ```html
  <button @click="openSocraticPopup(textSelection.text)" ...>
  ```

### 2. 改造 `InlineSocraticPopup.vue` 的加载逻辑
- **初始化静默状态**：
  在 `onMounted` 和 `watch(props.targetText)` 时，**不再调用** `fetchExplanation()` 进行首次启发式解释。
  初始化时，仅填充选中的上下文，`messages.value` 为空，展示一个清晰的等待提示及输入框。
- **舱内发送追问**：
  用户输入 `followUp` 并发送时，触发 `fetchExplanation(followUp)` 启动流式响应。
  智能体回复的内容会逐步 append 到 `messages.value` 列表中，形成舱内的对话历史。

### 3. 实现追问舱窗口自由拖拽与高透自适应毛玻璃
- **窗口拖拽定位**：
  在追问舱的 Header 上绑定拖拽鼠标事件，利用 absolute-style 绑定改变 `left`/`top`，使整个窗口可以在屏幕内被自由拖拽。
  移除全屏遮罩 `inset-0 bg-slate-950/20`，避免阻挡主聊天流。
- **高透玻璃外观**：
  融合亮/暗主题自适应的 `backdrop-blur-md` 毛玻璃设计，配备细致的阴影和微调圆角。

## 三、 验证方案

- **功能验证**：
  1. 划选文本点击小胶囊，确认右侧弹出悬浮舱窗口，且舱内**静默等待用户输入**（无自动 AI 启发文本）。
  2. 鼠标拖动舱体 Header，确认可以自由移动位置。
  3. 在追问舱内输入问题并发送，确认智能体的回复内容流式加载在**追问舱内部**，主对话流没有任何变化。
  4. 点击卡片上的 `💬 追问`，确认内容被引用到了主输入框上方，在主对话流里生成回复。
