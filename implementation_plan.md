## ⚠️ 架构级防冲突与回归安全防线 (Critical Constraints)

经深入全局扫描，成员 2 的重构需要特别防范与系统其他模块的耦合冲突，已确定以下**三处核心防线**：

### 1. `profile_api.py` 的静态 `KNOWLEDGE_DAG` 注入防御
*   **冲突现象**：`profile_api.py` 内部在模块加载时静态实例化了 `PATH_PLANNER = PathPlanner(KNOWLEDGE_DAG)`（使用本地硬编码字典）。这导致即使前端上传了新领域 PDF，路由端点调用的寻路引擎依然会被限制在写死的数学/ML 图中。
*   **防守策略**：在 `learning_strategy.py` 的 `PathPlanner` 的方法中引入**动态图谱热插拔融合机制**。
    ```python
    from rag_engine import graph_rag
    # 优先检测 graph_rag 是否有动态抽取的节点，有则自动覆盖 profile_api 传入的静态 dag
    active_dag = {node: list(prereqs) for node, prereqs in graph_rag.reverse.items()} if (graph_rag and graph_rag.nodes) else self.dag
    ```
    此举既不需要修改 `profile_api.py` 的引入逻辑，又能让 API 路由瞬间获得垂直领域的图谱泛化能力。

### 2. `models.py` 中的掌握度扩散 (LMCD) 依赖冲突
*   **冲突现象**：`models.py` 在执行掌握度扩散时，硬编码导入了 `from agent_swarm import DEFAULT_KNOWLEDGE_DAG` 并在 `diffuse` 算法中做参数传递。直接删除 `DEFAULT_KNOWLEDGE_DAG` 会导致 `models.py` 启动报错。
*   **防守策略**：保留 `agent_swarm.py` 中 `DEFAULT_KNOWLEDGE_DAG` 作为静态兜底和测试种子。但在 `bkt_engine.py` 内部的 `KnowledgeDiffusionEngine.diffuse()` 方法中进行**图谱重定向**：如果 `rag_engine.graph_rag` 拥有动态加载的垂直领域边，优先使用动态图谱进行掌握度多向扩散，彻底解决冷启动下的硬编码锁定。

### 3. `test_edumatrix.py` 回归测试套件兼容
*   **冲突现象**：现有的 `50 passed` 单元测试中，多处直接断言了 `DEFAULT_KNOWLEDGE_DAG` 下的节点跳转和 A* 路径结果。
*   **防守策略**：重构时，当 `rag_engine.graph_rag`中没有导入外部文档时，自动退回到 `DEFAULT_KNOWLEDGE_DAG` 的语义分支，从而确保所有既有的单元测试能够 `100% Passed` 完美通过，防止破坏回归基线。

## 🚀 第三部分：追求极致的“下一阶段（Phase 2）”前瞻优化方案

在实现上述基础智能化后，如果我们想在国赛终审中展示出**无可挑剔的学术创新性与产业落地性**，还可以在以下四个深度领域继续优化：

### 1. 记忆参数的“自校准”算法 (Dynamic Memory Curve Calibration)
*   **当前痛点**：SM-2 算法的参数（如默认易度因子 2.5、间隔天数倍率）对所有学生都是一刀切的。
*   **升格优化**：
    *   在 `anki_engine.py` 中引入一个轻量级的**在线梯度下降（Online Gradient Descent）或曲线拟合器**。
    *   根据学生历史所有的卡片复习打卡数据（时间、正确率、反馈延时），反向训练最适合该学生个体的遗忘半衰期参数，让遗忘曲线实现真正的“一人一曲线”，彻底干掉公式中的硬编码初始参数。

### 2. 跨学科流形的“投影对齐矩阵” (Cross-Domain Manifold Projection)
*   **当前痛点**：跨学科关联仅靠向量的 Cosine 相似度，无法区分“承接关系”和“纯相似关系”。
*   **升格优化**：
    *   在计算数学与物理/控制工程的关联时，不仅计算相似度，还学习一个**流形投影映射矩阵 (Manifold Alignment Projection Matrix)**。
    *   通过少量已知的锚点对（例如：偏导数-梯度，积分-面积），训练一个线性映射，将数学概念的向量空间映射到专业领域空间。这样，寻路引擎能够极其精准地回答：“学习 PLC 的 PID 调节时，最应该补齐数学中哪一部分微积分知识”，具备极高的学术解释性。

### 3. 规划路径拓扑的“规则可信度审计” (Topological Trust Audit)
*   **当前痛点**：大模型（Planner Agent）在重写路径或生成动作时，有可能因为幻觉，把后置知识排到了前置知识前面，违背了图谱的拓扑顺序。
*   **升格优化**：
    *   在 Planner Agent 输出最终 JSON 后，加入一个**无监督的拓扑合法性校验层 (Topological Checker)**。
    *   利用 NetworkX 自动检测生成的路径序列是否满足图谱有向无环图（DAG）的拓扑排序。如果发生“倒置（如先教神经网络再教链式法则）”，立刻进行拦截自愈，保证 AI 决策的 100% 安全与可信。

### 4. 学情足迹的“三维时间胶囊”回溯 (Time-Travel Cognitive History)
*   **当前痛点**：历史学情只是简单的静态列表，无法直观反映随时间演进的变化过程。
*   **升格优化**：
    *   在前端展示中，利用三维时间轴（3D Timeline），允许学生拖动时间滑块（例如回到“一周前”）。
    *   图谱和 3D 流形会动态还原到一周前的残缺状态，并对比展示过去一周通过 A* 寻路和 Anki 攻克了哪些红色的“重灾区”节点，极大地增强了人机交互的可观测性与成就感。

## 💎 第四部分：无人区封顶（Phase 3）工业级/学术级巅峰方案

如果您希望成员 2 负责的自适应路径和记忆引擎也跨入**“学术巅峰，拒绝一键开发”**的门槛，我们设计了以下四个极高壁垒的优化方向：

### 1. 基于图热传导与里奇流 (Ricci Flow) 的非欧空间动态测地线寻路
*   **理论深度**：传统 A* 算法只能在平坦静态的图上搜索，无法模拟概念之间的“阻尼与发散”物理规律。
*   **技术重构**：
    *   将动态知识图谱网络建模为微分流形。当学生在某个概念（如 PID 调节）卡关时，将掌握度亏空视为“冷源”，在此处引入**图热传导方程 (Heat Equation on Graphs)**。
    *   计算热力（挫败感/不理解度）随图谱边缘的扩散分布。
    *   利用**图里奇流 (Graph Ricci Flow)** 基于扩散反馈动态拉伸或压缩图的几何边缘权重（收缩难以逾越的鸿沟，拉近同质概念的距离），寻路引擎在此拉伸后的非欧流形空间上寻找**最小测地线 (Geodesic Path)** 作为推荐路径，具备颠覆性的图论理论深度。

### 2. 基于神经微分方程 (Neural ODEs) 与霍克斯过程的生成式间隔记忆
*   **理论深度**：Anki SM-2 将记忆衰减视作简单的离散天数，无法刻画大脑认知的连续衰减曲线。
*   **技术重构**：
    *   使用 **神经常微分方程 (Neural ODEs)** 替代 SM-2 的离散代数公式，将学生的记忆保持率 $h(t)$ 建模为连续的动力学系统：
        $$\frac{dh(t)}{dt} = f_{\theta}(h(t), t, \text{Cognitive\_Load})$$
    *   结合**自组织时序霍克斯过程 (Hawkes Process)** 拟合学生复习打卡行为的自激触发概率。
    *   系统不单推荐复习日期，而是计算出学生记忆曲线跌破 60% 临界斜率的**精确毫秒时间**，并在该时间节点动态生成“多模态视觉线索闪卡”以最小的认知代价唤醒长时记忆。

### 3. 基于纳什议价博弈论的 Swarm 多约束共识路径合成
*   **理论深度**：路径不仅是 A* 寻路加 LLM 润色，而是多个利益冲突的 Agent 之间博弈后的共识输出。
*   **技术重构**：
    *   在规划路径时，构建一个**纳什议价博弈模型 (Nash Bargaining Swarm Game)**。
    *   `Planner Agent` 追求最大知识点覆盖（争取多学新概念），`Diagnostician Agent` 追求最小学生挫败感（争取回退复习），`Director Agent` 追求资源多样性（代码/视频/公式混合）。
    *   三方 Agent 根据大模型推理，在博弈边界上进行多轮在线出价（Bargaining），最终收敛到 **Pareto 最优解（帕累托最优寻路路径）**，实现真正多智能体高度协同决策，摒弃粗暴的 Prompt 指令。

### 4. 结合 AST 节点控制流的“渐隐式”代码卡片自适应生成 (Faded Worked Examples)
*   **理论深度**：复习编程闪卡时，简单的文字背面无法锻炼手写代码的代码直觉。
*   **技术重构**：
    *   对于编程类卡片，当判定学生卡关时，`Visualizer Agent` 配合沙箱分析学生出错代码的 AST（抽象语法树）。
    *   定位到 AST 树中学生编写错误的控制流节点（例如循环边界或递归出口）。
    *   大模型动态重写该卡片的背面，不直接给答案，而是将其重构成一个 **“渐隐式代码填空题 (Faded Worked Example)”**，将 AST 出错节点对应的代码挖空，让学生通过渐进式的填空练习重新掌握该概念。
