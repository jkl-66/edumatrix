# EduMatrix（智教矩阵）终极融合系统架构报告

## 0. 执行摘要

EduMatrix 面向“中国软件杯”A3 教育智能体赛道，采用 **1+3+5 Swarm 智能体网络** 与 **GraphRAG + VisRAG + DRAG + InstructRAG + 双曲流形对齐** 的全链路 RAG 架构，目标是把教育资源生成从“单次大模型问答”升级为“可检索、可辩论、可校验、可回滚”的工业化流水线。

本仓库已经实现一套离线可运行的工程骨架：

- `app/main.py`：FastAPI 后端总挂载入口，初始化连接池、注册全局中间件并统一引入路由。
- `stream_api.py` / `profile_api.py` / `quiz_api.py` / `report_api.py` / `animation_api.py` 等：以独立路由形式提供高并发 SSE 流式输出、画像动态更新、自适应评测（CAT）、PDF 报告导出以及本地动画 Range 分片推流服务。
- `concurrency.py`：系统高可用限流熔断与异步线程工作池控制器。
- `rag_engine.py`：GraphRAG 知识边界召回 + VisRAG 图像切片/文本证据混合检索。
- `drag_debate.py`：正方、反方、法官三角色证据清洗，剔除低相关或矛盾证据。
- `instruct_rag.py`：面向生成 Agent 的证据约束模板，生成前保存内部审计 rationale。
- `manifold_alignment.py`：使用 Poincare Ball 双曲距离和规则冲突检测进行跨模态一致性校验。
- `agent_swarm.py`：完整 1+3+5 Agent 编排，支持资源并发生成、对齐失败回滚和量化评估闭环。
- `models.py`：证据驱动动态学生画像，声明式 ORM 与内存状态双轨并网，支持 10 维画像、不会原因占比与情感计算。
- `llm_client.py`：科大讯飞星火 WebSocket 适配器与离线 deterministic fallback。
- `bkt_engine.py` / `anki_engine.py`：Bayesian Knowledge Tracing (BKT) 掌握度估计引擎、Ebbinghaus 记忆遗忘衰减模型与 SM2 算法排程闪卡管理器。
- `app/utils/rl_planner.py` / `app/utils/recommendation_engine.py`：强化学习自适应路由调度决策器与五维资源战术编译器，支持破冰/实操/防忘/探究/跨界五大战术路线。
- `code_exec_api.py`：独立子进程安全隔离代码沙箱，限制 CPU 与内存防逃逸。
- `note_engine.py` / `learning_strategy.py`：智能笔记与错题反思提炼引擎，自适应 ZPD 教学策略规划器。
- `document_parser.py` / `ingestion.py`：多模态课件文档解析器（PDF/docx/txt/LaTeX）与 GraphRAG 拓扑图谱索引入库管道。
- `frontend/src/api/animations.js`：本地动画视频 Range 分片推流前端 API 适配。
- `observability.py`：智能体执行链路可观测性遥测系统（Trace, Metrics）。
- `swarm_orchestrator.py` / `run.py`：服务启动入口与命令行测试/仿真调度器。

> 答辩口径建议：不要承诺数学意义上的“绝对零幻觉”。更稳的表达是：EduMatrix 通过证据绑定、辩论过滤、生成约束和一致性回滚，把幻觉从不可见的模型内部错误，转化为可审计、可定位、可修复的工程质量问题。

## 1. 设计原则

### 1.1 幻觉的工程定义

在教育 RAG 中，幻觉不是单一问题，而是四类信息失真：

1. **边界缺失**：只召回知识碎片，遗漏前置概念或后续应用。
2. **证据污染**：检索结果夹杂弱相关、过时或矛盾内容。
3. **生成漂移**：模型没有严格绑定证据，输出听起来合理但缺少依据。
4. **跨模态错配**：讲义、代码、导图、视频脚本表达的逻辑不一致。

因此 EduMatrix 不把防幻觉押在一次 Prompt 上，而是在摄入、检索、清洗、生成、校验、反馈六个环节都设置质量闸门。

### 1.2 不盲目堆技术

本方案把前沿架构拆成可落地职责：

- GraphRAG 负责课程概念边界与依赖路径。
- VisRAG 负责保留公式、图表、伪代码等不可降维视觉信息。
- DRAG 负责证据清洗，不让噪声进入生成上下文。
- InstructRAG 负责生成前的证据绑定和输出约束。
- 双曲流形对齐负责讲义、代码、导图、视频脚本的交付前一致性校验。

## 2. 1+3+5 Swarm 智能体矩阵

### 2.1 交互中枢层：1 个前台 Agent

**Agent 1：苏格拉底导师（Socratic Tutor）**

- 职责：系统唯一学生入口，完成多轮对话、意图识别、Handoff 路由。
- 不做的事：不直接生成长讲义、不越权写代码、不绕过校验下发资源。
- 工程实现：`agent_swarm.py` 中的 `EduMatrixSwarm.process()` 是入口，后续可替换为 WebSocket/HTTP 服务。

### 2.2 认知治理层：3 个后台 Agent

**Agent 2：画像探针（Profile Probe）**

- 隐式更新 10 维动态画像：知识基础、易错点与误概念、理解-熟练-迁移、认知加工与负荷、学习策略、元认知、动机目标、情绪韧性、互动偏好、学习情境。
- 当前实现：`StudentProfile.update_from_message()` 从学生自然语言中解析专业、目标、薄弱概念、不会原因、互动偏好与情绪信号；`StudentProfile.update_from_feedback()` 接收学生反馈、正确率、自评置信度和提示次数，完成随学随新。
- 每条画像判断都保留 `ProfileEvidence`，包含来源、原始文本、特征、权重、置信度和时间戳，避免把学生永久标签化。
- 系统会把“因为什么不会”拆成可计算占比：前置知识缺口、误概念/易混点、认知负荷过高、学习策略不足、自我判断失准、情绪与信心阻滞、讲解方式适配需求。

**Agent 3：路径规划师（ZPD Planner）**

- 基于画像与 GraphRAG 规划最近发展区（ZPD）学习路径。
- 当前实现：`ZPDPlannerAgent.plan()` 调用 `HybridRAGPipeline.retrieve()`，返回图谱路径和证据包。

**Agent 4：量化评估师（Effect Evaluator）**

- 回收练习正确率、代码沙盒错误率、停留时长。
- 若正确率低或沙盒错误率高，触发重规划与难度降级，并把学习反馈写回学生画像。
- 当前实现：`LearningSignal.needs_replan` 提供闭环判断接口。

### 2.3 资源兵工厂：5 个生成 Agent

**Agent 5：理论教授（Theory Professor）**

- 生成专业 Markdown 讲义。
- 必须引用 DRAG 后的干净证据。

**Agent 6：逻辑画师（Logic Mapper）**

- 生成 Mermaid 思维导图。
- 节点必须与讲义术语一致。

**Agent 7：极客助教（Geek Assistant）**

- 生成 Python/PyTorch 代码实操案例。
- 必须与讲义中的公式、变量和池化类型一致。

**Agent 8：考官智能体（Quiz Master）**

- 生成选择、填空、代码改错等练习题。
- 每题附答案和解析，用于量化评估闭环。

**Agent 9：虚拟导演（Virtual Director）**

- 生成虚拟人视频脚本、TTS 文案和画面分镜。
- 生产环境对接科大讯飞虚拟人合成和语音合成 API。

## 3. 四重防幻觉 RAG 防线

### 3.1 第一重：GraphRAG + VisRAG 保证内容完整

**GraphRAG 管宏观边界**

`rag_engine.py` 中的 `GraphRAG` 维护课程概念 DAG。例如“池化层”的召回路径会覆盖：

`卷积核 -> 填充 -> 步长 -> 卷积运算 -> 特征图 -> 池化层`

这避免模型只解释池化层定义，却漏掉它为什么接在特征图之后、与卷积核/步长有什么关系。

**VisRAG 管微观视觉信息**

`VisRAG` 使用图像切片元数据作为生产向量库的本地替身。对于公式、矩阵、拓扑图、伪代码，系统保留 `raw_image_ref`、页码、标签和语义锚点。生产环境可替换为：

- PDF 渲染切片：PyMuPDF / PaddleOCR 仅做版面定位，不把公式强行 OCR 成文本。
- 多模态向量化：本地视觉编码模型、bge-visualized、CLIP 系列或企业内多模态 embedding。
- 向量库：Milvus、FAISS、pgvector。

### 3.2 第二重：DRAG 多智能体辩论保证信息纯净

`drag_debate.py` 实现 DRAG 风格的三角色裁决：

- **正方 Agent**：判断证据是否能回答学生问题。
- **反方 Agent**：寻找不相关、矛盾、池化类型混淆等风险。
- **法官 Agent**：综合检索分、正方分、反方风险，决定保留或剔除。

输出为 `DebateVerdict`，每条证据都有可解释分数和裁决理由。这样答辩时可以展示：系统不是把 top-k 检索结果直接塞给模型，而是先经过证据级质量审判。

### 3.3 第三重：InstructRAG 保证逻辑严密

`instruct_rag.py` 的生成器在每个 Agent 生成前构造三部分约束：

1. GraphRAG 学习路径，保证前置逻辑不丢。
2. DRAG 后的证据列表，保证输出有来源。
3. 学生画像和纠偏要求，保证个性化且可回滚。

系统保存 `private_rationale` 供审计和回滚使用，但不把内部推理草稿直接展示给学生。这比在前端暴露 `<think>` 更安全，也更符合生产系统的可审计设计。

### 3.4 第四重：双曲流形对齐保证跨模态自洽

`manifold_alignment.py` 做两类校验：

- **双曲距离**：把讲义、代码、导图映射到 Poincare Ball，计算测地线距离。
- **符号级冲突检测**：例如讲义说“最大池化”，代码却使用 `nn.AvgPool2d`，即使向量距离不高，也会被强制拦截。

若 `AlignmentReport.passed == False`，`EduMatrixSwarm.process()` 会携带失败原因回滚到资源生成 Agent，最多重试 `EDUMATRIX_ROLLBACK_LIMIT` 次。

## 4. 端到端执行流

以学生问题为例：

> 我还是看不懂卷积神经网络里的池化层，能用图和 PyTorch 代码演示最大池化吗？

执行过程：

1. 苏格拉底导师接收请求。
2. 画像探针更新画像：记录“池化层”为薄弱点，识别为视觉/代码导向。
3. 路径规划师调用 GraphRAG，召回池化层前置路径。
4. HybridRAG 调用 VisRAG 和文本知识库，得到候选证据。
5. DRAG 正反方法官裁决证据，形成干净上下文。
6. 五个资源 Agent 并发生成讲义、导图、代码、练习题、视频脚本。
7. 双曲流形对齐检查跨模态一致性。
8. 量化评估师生成学习信号，为下一轮路径规划提供反馈。
9. 前端下发“讲义 + Mermaid 导图 + 代码沙盒 + 练习题 + 虚拟人脚本”的多模态资源包。

### 4.1 对话式学习画像自主构建实现

EduMatrix 已把“对话式学习画像”落到 `models.py` 和 `agent_swarm.py` 的运行链路中，不再依赖传统表单。系统把学生每一次自然语言表达、诊断反馈和学习结果转化为可审计证据，再聚合成动态画像和教学动作。

**1. 证据账本**

`ProfileEvidence` 是画像的最小证据单元，字段包括：

- `source`：证据来源，区分学生自然语言、学生反馈、表现信号。
- `text`：原始表达，保留“我为什么不会”的上下文。
- `features`：从文本与反馈中抽取出的学习状态特征。
- `weight`：该证据对画像的影响权重。
- `confidence`：该证据可信度。表现反馈高于单纯自述，避免学生自述不准导致画像漂移。
- `timestamp`：用于后续做最近性加权和随学随新。

**2. 十维动态画像**

`StudentProfile.dimension_states` 维护 10 个维度，每个维度都包含分数、置信度、状态描述、证据数量、证据片段和推荐干预：

| 维度 | 画像回答的问题 | 直接驱动的 agent 行为 |
|---|---|---|
| 知识基础 | 当前哪些概念会、哪些不会 | 补前置知识、生成微诊断 |
| 易错点与误概念 | 为什么反复错 | 反例辨析、相似概念对照 |
| 理解-熟练-迁移 | 是看懂、会做，还是能迁移 | worked example 后接变式题 |
| 认知加工与负荷 | 是否被长题干、多条件、多步骤压垮 | 拆步骤、列条件清单、降低一次性信息量 |
| 学习策略 | 是否只看答案、缺少检索练习和复盘 | 提示阶梯、错因复盘、间隔复习 |
| 元认知与自我调节 | 自评和真实表现是否一致 | 题前自评、题后校准 |
| 动机目标与意义感 | 学习目标是否明确，与专业/考试/项目如何连接 | 用专业场景解释知识，给学生选择权 |
| 情绪信心与韧性 | 是否因挫败、焦虑、低信心影响学习 | 小步成功体验，反馈聚焦下一步行动 |
| 互动与表达偏好 | 什么讲解方式更有效 | 切换图示、代码、例子、追问或视频脚本 |
| 学习情境与公平支持 | 可用时间、专业背景、课程要求是否明确 | 按现实约束重排学习计划 |

**3. “因为什么不会”的占比计算**

系统把不会原因拆成 7 类 `LearningStateCause`：

- 前置知识缺口
- 误概念/易混点
- 认知负荷过高
- 学习策略不足
- 自我判断失准
- 情绪与信心阻滞
- 讲解方式适配需求

`StudentProfile._refresh_dynamic_profile()` 会对所有证据做加权聚合：

```text
原因原始分 = 证据权重 × 证据置信度 × 最近性权重 + 行为状态补偿
原因占比 = 单项原因原始分 / 所有活跃原因原始分之和
```

例如学生说：

> 我是计算机专业，期末要考。池化层看不懂，最大池化和平均池化总混，题干长会漏条件，复习只会看答案，我以为会了一做就错，希望你用图和代码一步步提示。

画像探针会抽取：

- 专业：计算机专业。
- 目标：期末复习。
- 薄弱点：池化层。
- 互动偏好：图示演示、代码实操、分步引导。
- 不会原因：误概念、认知负荷、策略不足、元认知失准、讲解方式适配需求、前置知识缺口。

随后输出归一化占比，例如：

```text
误概念/易混点: 21.7%
认知负荷过高: 20.4%
学习策略不足: 17.9%
自我判断失准: 16.8%
讲解方式适配需求: 13.2%
前置知识缺口: 10.0%
```

占比不是装饰性指标，而是直接改变教学动作：误概念占比高则优先做最大池化/平均池化反例辨析；认知负荷高则先列题目条件清单；策略不足则减少直接给答案，改成提示阶梯和检索练习；元认知失准则每题前后做信心校准。

**4. 随学随新与多轮对话记忆**

*   **自适应画像更新入口**：
    - `update_from_message(message)`：处理自然语言对话，隐式抽取学生专业背景、学习目标、困惑概念、反馈偏好和焦虑/挫败情绪。
    - `update_from_feedback(feedback, accuracy, self_confidence, hint_count)`：处理学习后的真实反馈与答题结果，定量更新掌握度、策略缺口和元认知校准。
    - 当 `EffectEvaluatorAgent` 检测到答题正确率低或沙盒错误率高时，系统会把重规划信号写回画像，使下一轮资源生成自动降级或补充前置概念。
*   **滑窗多轮对话上下文记忆系统**：
    - 在 [stream_api.py](stream_api.py) 中持久化维护对话历史，并在用户提交后续请求时通过滑窗策略保留最近多轮上下文，送入统一 LLM 提示词进行渲染。
    - 结合 `_resolve_coreference` 算法对“它、这个、为什么是这样”等指代不明口语进行消解自愈，支持连续深度追问，打破“单轮对话无记忆”瓶颈。

**5. 学情报告双轨分流生成（温情叙事 vs 理性诊断）**

为了适配不同页面的阅读语境，画像探针在画像更新时会同时触发双轨报告并行生成（采用 `asyncio.gather` 异步并发以优化响应耗时）：
- **温暖鼓励的成长信笺（供学习画像页使用）**：
  - 调用 `Storyteller Agent`，用温和体贴的语气肯定进度，并将掌握度与不会原因通过比喻形式（如“种子的萌发”、“地基的加固”、“背包精简行李”）轻柔道出，侧重于心理疏导与主动建构引导。
  - 数据持久化在 `student_profiles.narrative_report` 字段中。
- **严肃理性的诊断报告（供首页仪表盘使用）**：
  - 调用 `Dashboard Agent`，字数精炼（250-350字），采用客观、中立、科学的学术诊断口吻，详细解释学情状态背后的成因与策略推导。
  - 🚫**核心防污染规范**：通过正则表达式彻底剔除特殊表情/图标符号，清洗掉多余的段落空行，排版极其清爽好看。
  - 数据持久化在 `student_profiles.dashboard_report` 字段中。

**6. 画像进入生成链路**

`InstructRAGGenerator._build_plan()` 已把 `profile.profile_prompt()` 和 `profile.state_report()` 注入每个资源 Agent 的提示词。理论教授、逻辑画师、极客助教、考官智能体和虚拟导演不再只拿到 RAG 证据，还会拿到学生当前 10 维画像、不会原因占比、证据片段和推荐干预。因此同一个“池化层”主题会根据学生状态改变输出：

- 前置缺口高：先补卷积核、步长、特征图。
- 误概念高：先做最大池化与平均池化对照。
- 认知负荷高：讲义分成小步，导图减少分支。
- 策略缺口高：练习题加入检索、复盘和间隔复习。
- 元认知失准高：每道题加入自评和题后校准。
- 情绪阻滞高：降低首题难度，反馈只指向任务和下一步。

**7. 学生可解释输出**

`render_console_summary()` 会在资源包前展示 `profile.state_report()`，包括每个维度的分数、置信度、证据数量、证据片段，以及不会原因占比。这保证系统不仅“悄悄个性化”，还可以向学生和教师解释：为什么现在先补这个知识点，为什么这一轮不是直接给答案，为什么安排反例题或复习题。

**8. 强化学习自适应资源调度决策 (Q-Learning)**

传统的自适应引擎通常依赖纯规则（如：若掌握度低则推送讲义，若掌握度高则推送测验），但这难以适应学生复杂的实时动态心理状态变化。因此，本系统设计并实现了基于强化学习 Q-Learning 的自适应资源推荐算法：
- **状态空间离散化 (State Space)**：将连续的三维动态指标进行离散化编码：
  - 掌握度 $P(\text{Mastery})$：划分为 LOW ($<0.35$)、MID ($[0.35, 0.75]$)、HIGH ($>0.75$)。
  - 认知负荷 $L_{\text{cognitive}}$：划分为 HIGH ($\ge 0.6$)、NORMAL ($<0.6$)。
  - 挫败度 $F_{\text{frustration}}$：划分为 HIGH ($\ge 0.4$)、LOW ($<0.4$)。
  这产生 12 种典型心智学情状态组合，构成了 Q-table 的行键。
- **动作空间 (Action Space)**：包含 5 大物理教学动作，即 `ACTIONS = ["lecture", "mindmap", "code", "quiz", "video"]`。
- **即时奖励计算 (Reward Function)**：综合评估学习效益与心智消耗：
  $$R = (\text{mastery}_{\text{after}} - \text{mastery}_{\text{before}}) \times 100 + \text{quiz\_bonus} - (5.0 \times \text{frustration} + 2.0 \times \text{load})$$
  当答题正确时，`quiz_bonus` 为 $+10$；答错时为 $-10$；闲聊或非评测交互时为 $0$。该公式平衡了“能力增长”与“情绪保护/负荷过载”之间的冲突，惩罚导致学生感到高度挫败或工作记忆过载的推荐动作。
- **Q值时序更新与探索**：采用 Epsilon-Greedy 决策策略，以 $15\%$ 的探索概率进行随机动作探索，以 $85\%$ 的概率极可能选择 Q 矩阵中当前状态价值最高的动作类型。当所有动作价值均为 $0$ 时（冷启动），自动平滑回退至基于认知风格的静态启发式规则。

**9. 战术大纲编译器与五维自适应资源矩阵**

系统首创 **五维自适应资源矩阵** 视图（在 `Dashboard.vue` 和 `app/utils/recommendation_engine.py` 中实现），对任意核心概念（如池化层），均备齐了“讲义、思维导图、代码案例、练习题、视频脚本” 5 类资源：
- **个性化大纲修饰器**：根据学生的专业背景与心理状态，对资源的大纲进行针对性修饰。例如，若检测到学生为计算机科学专业，代码案例大纲会自动标记“代码采用规范工程级写法”；若学生挫败感高于 0.7，讲义大纲会自动追加“融入通俗比喻以降低负荷”。
- **战术生成路线 (Tactical Pathways)**：支持 5 大战术路线的一键切换与后台实时重编译：
  1. **破冰路线 (ICE_BREAKER)**：重点梳理前置依赖概念，降低讲解梯度。
  2. **实操路线 (PRACTITIONER)**：以完整的、带详细注释的算法代码演示为主，以干代讲。
  3. **探究路线 (EXPLORER)**：采用严密数理逻辑，大量运用 LaTeX 进行推导，侧重学术深度。
  4. **防忘路线 (RESCUE)**：使用 `[!WARNING]` 或 `[!CAUTION]` 在醒目位置标注核心易错混淆区，并提供抗遗忘口诀。
  5. **跨界路线 (FUSION)**：引入跨学科关联和多领域应用场景。
- **状态联动与本册持久化**：卡片与数据库中的 `DBNote` 状态实时联通。对于尚未生成的资源卡片，点击触发特定 Agent 进行后台并发流式生成，生成后写入 `DBNote`，将卡片状态标记为“已就绪”，实现真正的学情沉淀。

- **核心 RAG 检索**：验证 GraphRAG/VisRAG 是否能精准召回池化层的前置路径与图像切片上下文。

- **对话式动态画像**：验证画像探针是否能从自然语言中隐式解析出专业、目标、薄弱点、不会原因占比和 10 维动态画像状态。
- **认知状态闭环更新**：验证学生答题与诊断反馈是否能根据正确率、置信度与提示次数，增量更新掌握度、策略缺口与元认知状态。

- **垃圾词与指代自愈**：测试口语指代消解在“这这那那”等垃圾词打断和历史长上下文下的自愈还原能力。
- **自适应 ZPD 路径规划**：验证 BKT 状态下的 ZPD 教学档位判定、前置知识回滚计划以及二档路由（SIMPLIFIED/ADVANCED）降级分流。
- **高可用与沙箱安全**：测试熔断器（CircuitBreaker）状态扭转，以及代码沙箱（SandboxRunner）在运行超时或恶意循环下的资源限制硬截断。
- **强化学习自适应调度**：测试 Q-Learning 离散状态转换（get_state_key）、复合即时奖励（calculate_reward）计算、Q 矩阵的时序增量 TD 更新以及与画像库的持久化集成。
- **五维资源战术编译器**：验证在破冰/实操/防忘/探究/跨界五大不同战术路线下，推荐算法能够生成包含定制指导词的个性化大纲，并与 `DBNote` 完成已就绪/待生成状态的闭环同步。
- **高并发 PDF 与前端集成**：验证后台 BrowserPool 资源复用生命周期，以及前端 stores、致谢墙、Anki 闪卡、数字人嘴形滤波等核心组件的结构与字段一致性。

## 6. 生产部署升级路径

### 6.1 数据层

- **当前**：使用 **SQLite (WAL 预写日志模式) + SQLAlchemy 2.0 ORM** 承载 9 张业务实体表（学生画像、流形对齐日志、答题记录、复习计划、笔记、对话历史、代码沙箱执行日志等），保障事务级并发读写；向量和多模态图像检索基于 **本地轻量级 FAISS 索引**，进行极速的相似度匹配与序列化持久化。
- **生产**：
  - 关系与图数据：升级为 **PostgreSQL/RDS** 关系型集群以承载海量高并发业务，并引入 **Neo4j** 存储并查询课程知识图谱（GraphRAG）的实体关系。
  - 向量数据：将本地 FAISS 索引迁移至分布式向量数据库（如 **Milvus / Chroma / pgvector**）集群，支持动态增删索引和更大规模的特征向量检索。
  - 数据摄入：由离线脚本入库升级为自动化摄入管线，集成 **Apache Tika**、**PyMuPDF** 等解析 PDF/Office 文档，使用多模态视觉模型（如 **ColPali**）生成切片特征向量并对齐学科大纲。

### 6.2 模型层

- **当前**：统一抽象模型客户端 `llm_client.py`，默认使用本地离线 `DeterministicEducationLLM` 保证 CI 和单元测试顺利执行；生产环境已适配 **科大讯飞星火 API（WebSocket）** 与 **OpenAI-compatible 远程网关**。
- **生产**：
  - 星火模型集群：生产部署设置 `EDUMATRIX_LLM_PROVIDER=spark`，并使用配置完备的星火 V3.5 专业大模型集群作为核心 Agent 调度大脑。
  - 本地 vLLM 降级：配置高可用 **本地 vLLM 备用节点（Qwen2.5-Coder-32B）**，当星火 API 超时或网络异常时，通过 **CircuitBreaker（熔断器）** 机制自动切流降级。
  - 媒体渲染：虚拟导演（Virtual Director）输出的分镜与脚本直接对接 **讯飞 TTS 和虚拟人合成 API**，生成高保真微课短视频。

### 6.3 服务层

- **当前**：采用 **FastAPI 异步高并发架构** 启动 Web 服务，结合 `asyncio.gather` 在协程池中并发驱动 5 大资源生成智能体，通过 **SSE (Server-Sent Events) 实时流式通道** 与前端 Vue 3 单页面应用交互。
- **生产**：
  - 分布式任务网格：引入 **Celery / RQ**，将大模型重试、视频渲染合成、隔离代码沙箱执行等重计算逻辑剥离为后台异步任务，走 **Redis Stream / RabbitMQ** 消息队列进行任务分发与状态同步。
  - 容器化集群：在 Docker 容器化方案的基础上，使用 **Kubernetes (K8s)** 进行水平扩展与弹性伸缩，配合 Nginx 进行灰度发布和反向代理限流。

### 6.4 可观测性

- **当前**：基于 `observability.py` 本地记录调用链追踪（Trace ID, Span ID）、大模型延迟及 Token 消耗。
- **生产**：
  - 接入标准 **OpenTelemetry** 协议，打通 APM，将调用链日志自动吐出至 **Jaeger / Zipkin** 平台进行可视化分布式追踪。
  - 采集以下核心生产指标并配置 **Grafana 监控大盘**：
    - *检索质量*：Top-K 命中率、证据覆盖率、图谱节点偏离度、RAG 检索均分。
    - *辩论清洗*：DRAG 证据保留率、反方风险拦截率、平均法官评分。
    - *生成对齐*：各 Agent 耗时、对齐回滚拦截率、局部重写成功率。
    - *教规诊断*：贝叶斯掌握度准确度、遗忘衰减触发频次、错题反思命中率。
