# 产品术语与角色权限

版本：`M0-v0.1-draft`  
适用范围：EduMatrix 比赛闭环及后续企业培训扩展

## 1. 术语原则

- 同名对象必须只有一个业务含义；不同生命周期的对象不得复用同一状态字段。
- 页面文案、API Schema、数据库字段、事件和测试用例优先使用本表术语。
- “完成”“掌握”“胜任”“免修”是四个不同结论，不能互换。
- “生成内容”不自动等于“已审核资源”，“点击资源”不自动等于“掌握知识”。

## 2. 核心术语

| 术语 | 规范定义 | 唯一身份/版本 | 明确不等于 |
|---|---|---|---|
| 课程 `course` | 围绕明确培养目标组织的模块、章节、知识点、任务、资源和评测集合 | `course_id + version` | 单个上传文件、聊天会话 |
| 教材 `textbook` | 课程采用的具有稳定目录与版本的主要教学材料 | `textbook_id + edition` | 任意参考链接 |
| 文档 `document` | 被摄入系统的一个来源文件或网页快照，保留来源、许可、哈希与所有者 | `document_id + content_hash` | 分块、课程 |
| 章节 `chapter` | 课程结构中的有序教学单元，可锚定一个或多个来源范围 | `chapter_id + course_version` | 知识点本身 |
| 知识点 `knowledge_point` | 可被讲解、检索、练习和测量的最小稳定概念或技能单元 | `kp_id + gold_version` | 页面标签、关键词 |
| 产物 `artifact` | 由人或 Agent 生成并可版本化、审核、发布的学习成果，如讲义、指南、题组 | `artifact_id + version` | 临时流式文本 |
| 资源 `resource` | 学习者可消费或操作的已登记对象；可以引用一个已发布产物或原始材料 | `resource_id + version` | 未持久化的模型输出 |
| 任务 `task` | 有输入、责任人、状态、截止条件、输出和验收标准的工作单元 | `task_id` | 学习计划、后台线程 |
| 计划 `plan` | 尚可编辑和确认的学习安排，包含目标、约束和候选节点 | `plan_id + version` | 已发布路径 |
| 路径 `path` | 由已确认计划发布出的有向学习节点序列，允许按证据重规划并保留历史 | `path_id + version` | 推荐列表 |
| 路径节点 `path_node` | 路径中的可执行学习单元，绑定知识点、资源、任务与通过条件 | `node_id` | 章节标题 |
| 事件 `event` | 已发生、不可被就地改写的领域事实，携带时间、主体、对象、版本和追踪 ID | `event_id + schema_version` | 命令或当前状态 |
| 证据 `evidence` | 支撑技术、学习或胜任结论的可定位记录，含来源、等级、范围和完整性 | `evidence_id` | 宣传文字 |
| 画像 `profile` | 对学习者背景、目标、偏好、知识状态和限制的结构化估计 | `profile_id + snapshot_version` | 永久人格标签、单次问卷 |
| 画像快照 `profile_snapshot` | 某次诊断/生成/决策实际使用的不可变画像输入 | `snapshot_id` | 可变的当前画像 |
| 记忆 `memory_item` | 经来源、置信度、敏感级别和有效期管理的可复用上下文 | `memory_id + version` | 完整聊天日志 |
| 审核 `review` | 对产物、标准或证据作出的独立检查记录，可自动或人工执行 | `review_id` | 生成 Agent 自评 |
| 发布 `publication` | 将特定版本产物或标准变为指定受众可用的受控动作 | `publication_id` | 保存草稿 |
| 诊断 `diagnosis` | 基于画像与可观测证据识别当前差距、原因和不确定性 | `diagnosis_id` | 简单用户分类 |
| 学习完成 `completion` | 学习节点规定动作已执行并满足最低完成条件 | 节点证据 | 掌握、胜任 |
| 掌握 `mastery` | 对某知识点的当前能力估计，必须由有区分度的作答/实操证据更新 | `kp_id + estimate_time` | 阅读、点击、自述会了 |
| 胜任 `competency` | 在岗位标准指定条件下独立完成任务，并满足质量、安全、提示依赖和保持要求 | `standard_version + evidence_set` | 一次练习通过 |
| 免修 `exemption` | 基于可接受证据和审批策略豁免某培训模块，不自动豁免岗位责任 | `exemption_id` | 删除能力要求 |
| 金标 `gold` | 经版本冻结、可由评测复用的预期事实、标签、路径或结果 | `gold_version` | 模型临时答案 |
| 原子事实 `atomic_claim` | 能独立判定支持、矛盾、无依据或不可判定的最小事实陈述 | `claim_id` | 整段文本 |
| 追踪 ID `trace_id` | 串联一次闭环中请求、Agent、产物、审核、反馈与决策的关联标识 | UUID | 用户 ID、会话密钥 |
| 领域包 `domain_pack` | 可替换的垂直领域配置，包含本体、岗位任务、规则、模板、检查器和评测集 | `domain_id + version` | 在通用服务中散落的 if/else |

## 3. 角色模型

| 角色 | 核心责任 | 默认数据范围 | 关键禁止项 |
|---|---|---|---|
| 管理员 `admin` | 用户/角色、课程治理、系统配置、审计和合规操作 | 被授权组织或本地比赛实例 | 不得读取明文密钥；不得替代领域专家签核高风险胜任 |
| 课程教师 `teacher` | 维护课程、知识点、题目、资源，审核发布，查看授权学情 | 所负责课程和学生群体 | 不得访问无关课程私人数据；不得改写历史评测原始记录 |
| 助教 `assistant` | 答疑、初审、标注、运营和问题升级 | 被分配课程/班级的受限子集 | 默认无发布、删库、角色管理和高风险免修审批权 |
| 学生 `student` | 提供画像、学习、练习、反馈、管理个人数据 | 自己的数据及被发布的公共/课程资源 | 不得读取他人画像、历史、私有文档和审核记录 |
| 访客 `visitor` | 查看公开介绍或使用显式匿名演示 | 公开资源和临时匿名会话 | 不得访问个人画像、持久化私有资源、教师/管理功能 |

企业扩展中的“培训管理员、直属主管、领域专家、合规审批人”不是 M0 比赛角色的别名；它们在企业标准流程中作为责任人扩展，不能挤入教师权限后失去审计边界。

## 4. 权限矩阵

符号：`A` 管理/配置，`C` 创建，`R` 读取，`U` 更新，`D` 删除/归档，`V` 审核，`P` 发布，`-` 默认拒绝。

| 对象/动作 | 管理员 | 教师 | 助教 | 学生 | 访客 |
|---|---:|---:|---:|---:|---:|
| 用户和角色 | A/R/U | R（课程成员） | R（被分配成员） | R/U（本人有限字段） | - |
| 课程定义 | A/R | C/R/U/D | R/U（授权草稿） | R（已发布） | R（公开） |
| 文档与知识库 | A/R/D | C/R/U/D | C/R/U（授权） | C/R/U/D（本人） | R（公开） |
| 知识点金标 | A/R | C/R/U/V/P | C/R/U（草稿） | R（已发布） | R（公开子集） |
| 画像与快照 | A（合规授权） | R（课程范围） | R（必要字段） | C/R/U（本人当前画像） | 临时创建，不可跨会话查询 |
| 学习计划/路径 | R（治理） | C/R/U（课程范围） | R/U（辅导） | C/R/U（本人草稿/确认） | - |
| 产物草稿 | A/R | C/R/U/D/V | C/R/U/V（初审） | R（本人生成） | 临时预览 |
| 发布资源 | A/R/D（治理） | C/R/U/V/P | R/V（初审） | R | R（公开） |
| 测验与作答 | R（审计） | C/R/U/V | C/R/U（授权） | C/R（本人作答） | 临时演示 |
| 评测金标/运行 | A/R | C/R/U/V | C/R/U（标注） | R（个人结果摘要） | - |
| 审计日志 | A/R | R（自己课程） | R（自己操作） | R（涉及本人摘要） | - |
| 系统密钥 | 配置引用，不显示明文 | - | - | - | - |
| 高风险免修 | 流程管理，不单独批准 | 仅建议 | - | 申请 | - |

## 5. 权限判定规则

1. 后端是最终权限裁决点；隐藏按钮不构成授权控制。
2. 访问判定至少包含身份、角色、对象所有者、课程/组织成员关系、对象状态和操作类型。
3. 跨学生访问默认拒绝；教师只能访问课程责任范围，不因角色名称获得全库权限。
4. 匿名演示必须由显式 `EDUMATRIX_DEMO_MODE` 开启，数据与正式账号隔离并可过期清理。
5. 删除优先采用可恢复归档；隐私删除请求另走彻底清除流程，并保留不含正文的合规审计记录。
6. 权限拒绝不泄露对象是否存在；按威胁模型返回 403 或 404。
7. 所有发布、撤回、角色变更、导出、删除和高风险审批必须写审计事件。

## 6. 术语一致性人工检查

让产品、开发、测试各自解释“完成、掌握、胜任、免修”。合格结果必须分别对应：节点动作、知识点估计、岗位任务证据、培训政策审批。任何两词被解释为同义，M0 术语门禁失败。
