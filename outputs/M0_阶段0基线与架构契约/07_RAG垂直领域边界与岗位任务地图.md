# RAG 垂直领域边界与岗位任务地图

版本：`domain-rag-0.1`  
主领域：**RAG 应用开发、知识库构建与幻觉评测技能培训**

## 1. 为什么选择该领域

它同时连接 EduMatrix 已有的知识库、解析、检索、引用、辩论、代码实操和评测能力，能把“个性化培训”落到可操作的岗位任务：导入资料、构建索引、调参、检查引用、定位幻觉、修复检索链路并用评测集验收。它也适合从学生项目迁移到企业 AI 应用工程师/知识工程师的转岗培训。

## 2. 范围边界

### 纳入

- 文档/网页/Notebook 的来源登记、许可核查、解析、结构保留和分块。
- Embedding、关键词/向量/混合检索、过滤、重排、引用和知识图谱辅助。
- Prompt/context 组装、拒答、证据约束和多 Agent 审核。
- 检索质量、事实支持、幻觉、覆盖率和画像—资源适配评测。
- Python 本地实验、API 调用、日志/trace、故障诊断和安全边界。
- 面向岗位任务的讲义、实操指南、分阶测试题和结果报告。

### 排除

- 训练通用基础模型、建设通用搜索引擎、真实企业数据接入。
- 医疗、金融、法律等高风险领域的生产决策；这些只作为安全案例，不作为领域专家结论。
- 以模型榜单或单一问答分数代替完整岗位能力。
- 真实企业 HR、LMS、SSO 和商业客户运营系统。

## 3. 目标工具链

| 层 | M0 冻结工具/概念 | 训练要求 |
|---|---|---|
| 内容 | Markdown、PDF、PPTX、DOCX、Notebook | 能识别来源、许可和结构丢失风险 |
| 处理 | 解析、清洗、分块、metadata、去重 | 能说明每个转换为何不破坏证据 |
| 检索 | BM25/关键词、向量、混合、重排、过滤 | 能用固定 query/金标比较方案 |
| 生成 | system/user/context、引用、拒答、温度 | 能把回答事实绑定来源范围 |
| 评测 | Precision/Recall、支持率、幻觉率、覆盖率 | 能按冻结分母复现结果 |
| 系统 | FastAPI、SQLite、前端工作台、trace | 能定位一条请求的输入—输出—决策 |
| 实操 | Python、HTTP、JSON、测试和日志 | 能完成参数调整、故障分支和验收 |

## 4. 岗位任务地图

| 任务 ID | 岗位任务 | 输入 | 交付物 | 关键知识点 | 客观验收 |
|---|---|---|---|---|---|
| JOB-01 | 课程资料准入 | 文件、来源 URL、许可证 | 来源清单、哈希、许可结论 | KP-01/02/03 | 无许可或高风险材料不进入正式索引 |
| JOB-02 | 文档解析与结构保留 | PDF/PPTX/DOCX/MD/Notebook | 结构化 Markdown/块 | KP-04/05/06 | 标题、代码、表格和来源定位可回溯 |
| JOB-03 | 分块策略设计 | 长文档、章节、query 样例 | chunk 配置和实验记录 | KP-07/08 | 不跨章节污染；固定 query 检索到金标块 |
| JOB-04 | 索引构建 | chunks、metadata、embedding | 可查询索引 | KP-09/10/11 | owner/course/version 过滤生效 |
| JOB-05 | 检索调优 | query 集、候选结果 | top-k/混合/重排配置 | KP-12/13/14 | Recall@k/排序结果达到预设验收线（线下标注） |
| JOB-06 | 引用约束生成 | context、问题、回答模板 | 有引用回答或拒答 | KP-15/16/17 | 每个原子事实均有支持或明确不确定 |
| JOB-07 | 幻觉定位与辩论审核 | 候选回答、来源、裁决规则 | claim 检查、裁决、返工 | KP-18/19/20 | 无依据 claim 被拒绝/降级；有依据 claim 保留 |
| JOB-08 | 画像适配资源生成 | 三组画像、知识点、资源契约 | 三类差异化资源 | KP-21/22/23 | 资源难度/形式与画像标签一致 |
| JOB-09 | 评测集运行 | 50+ case、模型配置 | 原始输出、标注、指标 | KP-24/25/26 | 可按 run_id 复现，不删失败案例 |
| JOB-10 | 故障诊断与恢复 | 日志、trace、失败任务 | 原因、修复、重跑记录 | KP-27/28/29 | 断网/低置信/服务异常有可解释降级 |
| JOB-11 | 实操技能验收 | 调参任务、隐藏测试、故障注入 | 技能工件和验收结果 | KP-30/31/32 | 参数、状态、故障和最终结果均记录 |
| JOB-12 | 企业培训迁移 | 岗位标准、先验能力、证据 | 转岗差距与培训方案 | KP-33/34/35/36 | 必修/免修和胜任结论可审计 |

## 5. 先修和等级

| 等级 | 能力描述 | 推荐任务 |
|---|---|---|
| L0 | 能解释课程资料、query、来源和回答 | JOB-01/02 |
| L1 | 能完成解析、分块、索引和基础检索 | JOB-03/04 |
| L2 | 能基于金标调优检索、引用和拒答 | JOB-05/06/07 |
| L3 | 能独立设计评测、调试失败、解释指标 | JOB-09/10 |
| L4 | 能按岗位标准迁移、审核和指导他人 | JOB-11/12 |

先修：Python/JSON/HTTP 基础、统计中的分母分子、基本机器学习概念。缺少先修时路径先进入补强模块，不用“直接生成更多高级材料”掩盖基础差距。

## 6. 主领域的成功定义

M0 只冻结定义，不宣称达标。后续系统若要声明“掌握 RAG 调优”，至少要有：固定数据版本、可复现配置、检索和生成结果、引用检查、故障恢复和延迟复测。单次聊天成功、页面出现引用或资源数量不构成岗位胜任。
