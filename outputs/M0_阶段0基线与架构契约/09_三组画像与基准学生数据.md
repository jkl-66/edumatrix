# 三组画像与基准学生数据

版本：`baseline-profile-0.1`  
数据性质：**全部为合成画像，不是真实学生数据**。机器可读数据见 `datasets/baseline_profiles.json`。

## 1. 画像设计目的

三组画像使用同一课程、同一知识点金标和相同业务目标，用来验证系统是否能在不改变事实标准的前提下调整先修路径、解释深度、资源形式、脚手架、题目难度和验收方式。画像不是人格标签；所有维度都应带来源、置信度和更新时间。

## 2. 对照表

| 项目 | 零基础转入 | 理论强、实操弱 | 有经验、需进阶 |
|---|---|---|---|
| ID | `profile-m0-beginner` | `profile-m0-theory` | `profile-m0-advanced` |
| Python/ML | 初识/不了解 | 中等/扎实 | 熟练/熟练 |
| RAG 经验 | 无 | 仅概念 | 做过基础向量问答 |
| 首要差距 | 术语、JSON、来源与链路 | 参数操作、隔离和故障定位 | 评测口径、恢复、企业标准 |
| 资源难度 | L0-L1 | L1-L2 | L2-L4 |
| 脚手架 | 高 | 中 | 低 |
| 讲义 | 术语、图示、短例 | 压缩理论、突出边界 | 权衡、迁移、风险 |
| 实操 | 复制后逐步改 | 控制变量和故障注入 | 开放调优和隐藏验收 |
| 题目 | 识别、单步应用 | 应用和调试 | 综合故障和决策 |
| 不应发生 | 直接推高级评测 | 重复大段基础 | 只给 hello-world |

## 3. 基准学生

M0 的主基准学生是 `student-m0-001`，对应零基础画像。选择它是因为最容易暴露系统是否真正处理先修、解释和失败恢复，而不是只对熟练用户生成漂亮答案。`student-m0-002/003` 用于同题对照和企业转岗扩展。

基准起始记录包括：背景、目标、时间约束、初始掌握度、证据、预期第一条路径和成功后的最小变化。后续自动测试不得依赖数据库中某个手工修改账号的当前状态，应从版本化 JSON 创建隔离测试数据。

## 4. 画像更新规则

| 证据 | 权重原则 | 允许的更新 |
|---|---|---|
| 自述/问卷 | 低到中，存在高估和理解偏差 | 设置初始先验和偏好，不直接判掌握 |
| 有区分度测验 | 中到高，需题目质量和难度信息 | 更新目标知识点掌握概率 |
| 代码/交互隐藏测试 | 高，绑定任务和环境 | 更新技能证据和提示依赖 |
| 教师/专家签核 | 高，但必须记录范围和版本 | 支持高等级能力结论 |
| 点击/停留/重复打开 | 低 | 只做参与信号，不单独提升掌握 |
| 延迟复测 | 高 | 更新保持和遗忘风险 |

每次生成、审核和动态决策绑定不可变画像快照；当前画像变化不能改写旧任务实际使用的输入。

## 5. 人工验收

对三组画像提交同一个“优化混合检索 top-k”目标。人工检查：核心事实与验收标准一致；三组的先修、解释、脚手架和难度存在可说明差异；差异能够反查具体画像字段；模型随机措辞变化不算适配证据。
