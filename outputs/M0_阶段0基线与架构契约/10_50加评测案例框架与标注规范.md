# 60 个评测案例框架与人工标注规范

版本：`evaluation-case-m0-0.1`  
机器可读案例：`datasets/evaluation_cases.jsonl`

## 1. 案例分布

| 类别 | 数量 | 主要风险 |
|---|---:|---|
| 知识准确性 `knowledge_accuracy` | 10 | 概念错误、过度绝对、事实边界 |
| 检索 `retrieval` | 10 | 召回、排序、来源、owner/版本隔离 |
| 生成 `generation` | 10 | 引用、拒答、资源契约、失败降级 |
| 画像适配 `profile_adaptation` | 10 | 难度、先修、偏好、路径和免修 |
| 审核/决策 `quality_review/decision` | 10 | claim 裁决、版本、证据和分母 |
| 安全/恢复 `security_recovery` | 10 | 越权、秘密、超时、幂等、断线和索引 |
| **总计** | **60** | 不含简单重复请求 |

每个案例包含稳定 ID、类别、画像、目标、输入、金标摘要、知识点、难度和失败分支。后续增加案例只能升版本，不能静默替换导致历史指标不可复算。

## 2. 原子事实标注

先把回答拆成最小可判定事实，再逐项标注：

| 标签 | 定义 | 示例 |
|---|---|---|
| `supported` | 来源直接支持，语义和条件一致 | 来源写“top-k 增大会增加候选”，回答同义表述 |
| `contradicted` | 来源明确否定、数值或条件冲突 | 来源上限 20，回答写上限 200 |
| `unsupported` | 所有允许来源都没有支持 | 资料未提作者生日，回答给出生日 |
| `unknown` | 来源损坏、冲突未裁决或标注信息不足 | PDF 缺页导致无法判断 |
| `non_factual` | 建议、礼貌语、流程导航等非事实内容 | “你可以先导入资料” |

`contradicted` 与 `unsupported` 都计入幻觉分子；`unknown/non_factual` 不进入可判定事实分母，但必须报告数量，防止通过大量不可判定内容粉饰结果。

## 3. 画像适配标签

分别标注五个维度：

1. `prerequisite_fit`：是否尊重先修差距。
2. `difficulty_fit`：概念、任务和题目难度是否在目标区间。
3. `format_fit`：资源形式是否利用偏好但不删除必修能力。
4. `scaffolding_fit`：提示和步骤是否符合经验水平。
5. `assessment_fit`：验收是否能测目标技能且不过难/过易。

每维取 `fit / partial / unfit / unknown`。案例通过条件默认五维中无 `unfit`，且 `difficulty_fit`、`assessment_fit` 必须为 `fit`；如采用加权分数，权重须在运行前冻结。

## 4. 难度档

| 档位 | 能力表现 |
|---|---|
| L0 | 识别术语、跟随单步操作 |
| L1 | 在明确步骤下完成基础链路 |
| L2 | 独立选择参数、解释结果和常见失败 |
| L3 | 处理冲突、隔离、恢复和开放调优 |
| L4 | 设计标准、评测、迁移和审核他人方案 |

## 5. 双人标注流程

1. 标注者只看冻结的来源和标注手册，独立标注。
2. 系统计算逐标签一致率；分类标签推荐报告 Cohen's kappa，连续/等级分数报告适当一致性统计。
3. 冲突由第三名裁决人或领域责任人处理，保留两份原标和裁决理由。
4. 标注手册变化升版本，旧运行不回写。
5. 标注者不得看到系统宣传指标，以减少目标诱导。

M0 可以完成规范和模拟标注样例，但不能虚构已有两名真实领域专家完成签核。正式结果发布前必须补实际标注者和评审记录。

## 6. 运行记录

每次评测保存：`run_id`、提交哈希、工作树状态摘要、课程/知识点/案例版本、画像版本、模型 provider/model/temperature、检索参数、开始结束时间、逐例原始输出、自动检查、人工标注和失败原因。任何超时、生成失败和安全拒绝按预先案例规则处理，不从分母中临时删除。
