# 官方三项指标口径

版本：`metrics-m0-0.1`  
状态：**公式冻结草案；当前不声明已经达到赛题目标。**

## 1. 幻觉率

### 公式

```text
幻觉率 = (contradicted 原子事实数 + unsupported 原子事实数)
         / 可判定原子事实总数 × 100%

可判定原子事实 = supported + contradicted + unsupported
```

### 规则

- 单位是原子事实，不是整条回答；一条回答可包含多个事实。
- 只使用该案例允许的冻结来源，不能临时联网补证据。
- `unknown` 和 `non_factual` 不进入分母，但单独报告；若 `unknown` 比例过高，结果标为不可充分解释。
- 引用存在但不支持 claim，仍标 `unsupported/contradicted`。
- 无输出、超时或安全拒绝按案例预设：需要回答且无合法拒答时记“任务失败”，并同时报告任务完成率；不能假装零事实即零幻觉。

目标 `<5%` 只有在案例、来源、标注、分母和运行全部冻结并完成后才能判定。

## 2. 画像—资源难度适配准确率

### 公式

```text
适配准确率 = 通过画像适配判定的资源实例数
             / 进入适配评测的资源实例总数 × 100%
```

一个资源实例默认需要：`difficulty_fit=fit`、`assessment_fit=fit`，其余三维无 `unfit`。讲义、实操指南和分阶题组分别标注，不因为同属一个资源包而自动全部通过。

### 规则

- 三组画像都必须有案例，不能只选表现最好的画像。
- 生成失败、缺少最低资源类型按预设记不通过；不得从分母删除。
- 资源事实错误先触发质量失败，即使难度看似合适也不能计为通过。
- 同一内容仅改称呼、语气或长度，不构成有效个性化。

目标 `>=85%` 需在双人标注和一致性报告后判定。

## 3. 核心知识点覆盖率

### 公式

```text
核心知识点覆盖率 = 满足有效覆盖条件的核心知识点数
                   / 32 × 100%
```

分母固定为 `datasets/core_knowledge_points.json` 的 `kp-rag-0.1` 版本 32 个核心知识点。

### 有效覆盖条件

每个知识点至少满足：

1. 讲义中被实质解释、实操中被执行，或测评中被有效测量；仅关键词出现无效。
2. 有来源引用或明确的项目组原创标记。
3. 有语义检查确认内容确实对应该知识点。
4. 产物状态为审核通过的指定版本。

一个知识点多次出现仍只计 1；扩展和先修知识点单独报告，不进入分母。目标 `>=90%` 对当前版本意味着至少 29 个核心知识点满足条件（`29/32 = 90.625%`）。

## 4. 必须同时报告的辅助指标

为避免三项指标掩盖失败，报告还必须列出：任务完成率、合法拒答率、`unknown` 比例、引用可定位率、案例类别分布、各画像通过率、失败/超时数、模型/检索配置和标注一致性。

## 5. 禁止操作

- 运行后更换知识点分母或删除低分知识点。
- 删除失败、超时、拒答或安全案例以提高结果。
- 把自动字符串匹配当成人工事实标注的完整替代。
- 把历史文档中的数字当成本轮正式运行结果。
- 只报告百分比，不报告分子、分母和逐例证据。
