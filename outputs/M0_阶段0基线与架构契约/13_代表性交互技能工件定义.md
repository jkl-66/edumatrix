# 代表性交互技能工件与证据等级

版本：`skill-artifact-rag-tuning-0.1`

## 1. 基准任务

任务名称：**调优混合检索并保持数据隔离**。学习者使用冻结课程和 10 个固定 query，调整关键词/向量权重、`top_k` 和重排阈值，使证据召回和排序改善，同时保持 owner/course/version 过滤，且能处理低置信和索引版本错配。

该任务同时具备参数调整、状态变化、故障分支和客观验收，适合作为比赛中“可交互资源/技能培训”的代表工件。

## 2. 输入与受控参数

| 项目 | 约束 |
|---|---|
| 数据 | `course-rag-engineering-m0@0.1.0`，不得临时增加答案文件 |
| query | 10 个固定检索案例，含精确词、语义同义、冲突、版本和隔离 |
| 参数 | `keyword_weight`、`vector_weight`、`top_k`、`rerank_threshold` |
| 固定项 | owner/course/version 权限过滤、来源白名单、随机种子（可用时） |
| 输出 | 参数快照、候选、重排、引用、耗时、失败和 trace |

## 3. 状态与故障

```text
not_started -> configured -> running -> evaluated -> submitted
                         \-> failed -> diagnosed -> rerun
submitted -> passed/needs_retry
```

故障分支至少覆盖：低置信无证据、owner 过滤后无结果、索引版本不一致、重排服务不可用。学习者必须解释故障和选择的恢复动作，而不是只按“重试”按钮。

## 4. 客观验收

| 维度 | 最低条件 |
|---|---|
| 正确性 | 10 个 query 的金标证据可定位，不能用无来源回答替代 |
| 隔离 | 跨学生私有文档 0 泄露；版本查询不混入其他版本 |
| 改善 | 与基准参数相比，至少一个预定义检索指标改善且无关键指标越过退化阈值 |
| 可解释 | 能说明每个参数的影响、至少一个失败原因和恢复选择 |
| 可复现 | 保存数据/配置/代码版本、运行 ID 和原始结果 |
| 安全 | 不关闭权限过滤、不暴露秘密、不在不可信宿主直接执行危险代码 |

M0 不预填具体改善百分比；基准运行后再冻结可实现阈值，禁止为了让当前实现通过而反向降低标准且不留 ADR。

## 5. 技能证据等级

| 等级 | 名称 | 证据要求 | 可支持的结论 |
|---|---|---|---|
| SE-0 | 查看 | 打开资源或观看演示 | 接触过，不证明会操作 |
| SE-1 | 操作 | 在明确逐步指引下完成 | 能跟随步骤 |
| SE-2 | 提示后完成 | 有有限提示，能修正错误 | 初步应用 |
| SE-3 | 独立完成 | 无过程提示，满足公开验收 | 独立完成该任务版本 |
| SE-4 | 隐藏验收 | 通过未见 query/故障和安全检查 | 可迁移应用能力 |
| SE-5 | 延迟保持 | 一段时间后重新通过或解释变化 | 技能保持证据 |
| SE-6 | 专家签核 | 指定领域/安全责任人审核范围和版本 | 支持高风险/岗位胜任结论 |

等级不能跳跃推断：SE-0 不能写“掌握”，SE-3 不能自动写长期胜任，高风险模块没有 SE-6 不得免修。

## 6. 展示效果

界面应允许评委看到参数、候选证据、前后指标、失败提示和最终验收，而不是只显示终端文本。即使当前前端尚未实现完整工件，M0 契约也固定后续 M5/M6 的验收界面和数据字段。
