# 企业岗位、转岗、必修免修与胜任规范

版本：`enterprise-sim-m0-0.1`  
声明：以下组织、岗位和人员均为**模拟样例**，不代表真实企业客户。

## 1. 企业岗位样例

模拟组织：`org-demo-aitech`  
岗位族：AI 应用与知识工程

| 等级 | 岗位 | 主要任务 | 能力要求 | 安全责任 |
|---|---|---|---|---|
| E1 | RAG 应用助理工程师 | 来源登记、解析、分块、基础索引、按步骤评测 | KP-01~17、KP-21~24、KP-28、KP-32 基础档 | 不得绕过 owner/许可；高风险发布需复核 |
| E2 | RAG 应用工程师 | 混合检索/重排、引用质量、幻觉评测、故障恢复、指导 E1 | 全部 KP-01~32，重点 KP-18~20、25~31 | 对发布质量、隔离和恢复方案承担技术责任 |

## 2. 转岗黄金样例

来源岗位：数据分析师  
目标岗位：RAG 应用助理工程师 E1  
已有能力：Python、Pandas、基础统计、模型评估概念。  
预期可申请免修：Python/JSON 先修、基础统计指标；必须通过证据复核。  
待补差距：许可和来源、文档解析、分块、检索、引用、画像适配、trace 和数据隔离。

| 能力 | 初始判定 | 证据 | 培训决策 |
|---|---|---|---|
| Python/JSON | 可能具备 | 合成项目仓库+隐藏测试 | 通过后免修先修模块 |
| 统计指标 | 部分具备 | 合成测验 | 缩短但不免除幻觉率口径 |
| 许可/来源 | 缺失 | 无 | 必修 |
| 文档/索引 | 缺失 | 无 | 必修+实操 |
| 检索/引用 | 部分概念 | 自述低置信 | 诊断后分层学习 |
| 数据隔离 | 未证明 | 无安全证据 | 高风险必修，不自动免修 |

## 3. 企业标准生命周期

```text
draft -> review -> published -> replaced -> retired
```

发布版本不可原地修改。新版本必须说明变更原因、生效日期、受影响岗位/批次/证书和迁移策略；历史训练批次、Agent 任务和技能证据继续指向原版本。`replaced` 表示有新版本但历史仍有效，`retired` 表示不得用于新判定。

## 4. 必修与免修政策

| 模块风险 | 例子 | 可接受免修证据 | 审批 |
|---|---|---|---|
| 普通 | 术语、基础 UI | 有效测验、课程证书、近期项目 | 系统建议+教师确认 |
| 关键 | 分块、检索、引用、指标 | 隐藏测试、可复现项目、人工抽查 | 教师/岗位负责人 |
| 高风险 | 数据隔离、秘密、危险执行、发布质量 | 同环境独立实操+隐藏安全测试+有效期 | 指定安全/领域责任人；可规定禁止免修 |

免修只豁免培训活动，不删除岗位能力标准，也不自动授予胜任。证据有范围、版本和有效期；标准升级后可要求补训。

## 5. 岗位胜任口径

| 指标 | 定义 |
|---|---|
| 任务通过 | 对岗位规定任务达到质量、安全和完成条件 |
| 提示依赖 | 无提示/有限提示/逐步指引，作为能力强度而非附注 |
| 隐藏验收 | 对未见输入、故障和安全边界仍通过 |
| 延迟保持 | 规定时间后复测保持，或能解释环境变化 |
| 专家签核 | 指定责任人确认范围、标准版本和证据包 |
| 时间到胜任 | 从培训分配到首次满足完整标准的有效学习时间，不把等待/系统故障混入 |

E1 模拟胜任要求：关键任务至少 SE-3，高风险任务至少 SE-4 并有人审；E2 要求关键任务 SE-4、高风险 SE-5/SE-6，具体阈值由正式企业标准确认。

## 6. 企业扩展非目标

- M0/M1 不建设完整 HR 档案、薪酬、招聘、绩效和组织人事系统。
- 不虚构真实客户、合同、专家签字和真实 ROI。
- SSO、LMS、HRIS 接口只定义映射和模拟契约，不阻塞比赛核心链。
- 不让“企业版”变成换皮页面；扩展价值来自岗位标准、任务证据、转岗差距和版本治理。
