# 第二领域迁移预案：示波器测量与实验室电气安全

版本：`migration-scope-0.1`  
状态：M0 预案，不在当前阶段实现完整课程或宣称迁移成功。

## 1. 选择理由

第二领域选择“示波器测量与实验室电气安全”，它与 RAG 软件工程在本体、输入、操作和风险上明显不同：需要设备部件、波形、多模态图片、物理参数、操作顺序、接线错误和高风险安全责任。它可以检验 EduMatrix 是否真正使用可替换 `domain_pack`，而不是把 RAG 名词换成另一门课程标题。

## 2. 差异矩阵

| 维度 | RAG 主领域 | 示波器/电气安全 |
|---|---|---|
| 核心对象 | 文档、chunk、query、引用、模型 | 探头、通道、量程、耦合、波形、电路 |
| 任务 | 检索、生成、评测、恢复 | 接线、测量、调参、读数、故障排查 |
| 资源 | 文档/代码/检索结果 | 图示、交互面板、波形、检查表 |
| 风险 | 数据泄露、幻觉、危险代码 | 设备损坏、人身触电、错误接地 |
| 验收 | query 金标、引用、指标 | 读数误差、操作顺序、隐藏故障、安全检查 |
| 专家责任 | AI/知识工程审核 | 实验教师/电气安全责任人 |

## 3. 迁移实验步骤

1. 建立独立 `domain-oscilloscope-0.1`，不得在通用服务散落领域 if/else。
2. 冻结设备本体、20+ 核心知识点、岗位/实验任务和安全禁令。
3. 建立三组画像：零基础、理论电路强但设备弱、熟练者进阶故障诊断。
4. 定义代表任务：测量未知正弦信号频率/幅值并排查接地或触发错误。
5. 生成讲义、交互操作指南和分阶测试题，加入图像/波形证据。
6. 运行同一 Agent/资源/评测通用内核，记录需要修改的对象和代码。
7. 若迁移要求修改通用 API 之外大量硬编码，判定领域包隔离失败。

## 4. 成功判定

- 80% 以上通用对象/API/事件/评测运行代码无需修改（具体统计方法在迁移前冻结）。
- 领域差异通过 domain_pack 的本体、任务、规则、模板、检查器和评测集表达。
- 高风险操作必须人工责任人审核，系统不会依据普通历史成绩自动免修。
- 主领域课程、画像和指标结果不受第二领域导入污染。

## 5. 预期展示

评委可看到同一培训内核切换到完全不同的技能：从文本检索调参转为设备面板和波形操作；路径、Agent、证据等级和企业标准仍复用，而知识点、任务、交互和安全规则全部替换。这才构成领域可迁移证据。
