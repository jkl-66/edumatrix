# ADR-002：统一数据底座，先交付可验证比赛闭环

状态：`proposed`  
日期：2026-07-23

## 上下文

全面追平竞品会造成对象重复和不稳定；只做演示页面又无法形成长期可审计能力。

## 决定

采用方案 B 的统一数据底座和方案 C 的交付范围：M0 冻结对象/契约，后续先实现一条稳定黄金旅程，再按阶段扩展。

## 影响

新增功能必须映射到 20 能力域、正式对象和验收案例。允许短期不实现重型企业功能，但不允许用页面静态数据代替底层对象。

## 撤销条件

若统一对象设计阻塞比赛核心链超过阶段容忍时间，可通过 ADR 缩减一次交付范围，但不得拆成互不兼容的第二套对象。
