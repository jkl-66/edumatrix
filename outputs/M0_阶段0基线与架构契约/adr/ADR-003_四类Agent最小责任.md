# ADR-003：按四类最小责任审计 Agent

状态：`proposed`  
日期：2026-07-23

## 决定

业务审计固定诊断、领域生成、审核裁判、决策调度四类职责。当前 9 个 AgentSpec 作为实现角色映射到四类，不以 Agent 数量作为完成或创新指标。

## 理由

四类正好覆盖赛题闭环，并能明确输入、输出、责任和禁止项。审核必须独立于生成，决策不得改写原始证据。

## 风险

现有代码可能将 planner/evaluator/router 的边界混合；后续通过 `agent_run`、I/O Schema 和 RACI 逐步治理，不在 M0 大改代码。

## 撤销条件

只有出现无法归入四类且拥有独立输入/输出/责任的稳定职责时，才通过新 ADR 增加类别。
