# 第 1 章 来源、许可与课程边界

## 1.1 为什么先审来源

RAG 系统的回答可信度从来源开始。文件能下载、能解析或出现在搜索结果中，不等于项目拥有再分发、修改和打包展示的权利。工程师必须把来源 URL、仓库/作者、版本、抓取时间、文件哈希、许可证和允许动作登记为数据，而不是只把文件复制到一个目录。

## 1.2 许可检查最小流程

1. 找到仓库级许可证和文件级声明。
2. 区分代码、文档、图片、数据集和第三方嵌入物的不同许可。
3. 判断当前动作是本地研究、比赛提交、修改、再分发还是商业使用。
4. 对无法确认的材料标记 `REVIEW_REQUIRED`，不进入正式预置库。
5. 在界面和作品包中保留署名、原始链接、许可证全文或链接及修改说明。

## 1.3 课程边界

本课训练 RAG 应用工程和评测，不训练基础模型，也不对医疗、金融、法律问题给出专业结论。示例资料只用于说明检索、引用和评测机制；回答必须根据来源证据或明确拒答。

## 1.4 实训检查

给出三个文件：一个有明确 CC 许可证、一个只有仓库 README 声明、一个许可证不明。学习者应输出准入表，并解释为什么第三个不能直接放入作品包。验收点：结论、证据位置、哈希和风险动作齐全。
