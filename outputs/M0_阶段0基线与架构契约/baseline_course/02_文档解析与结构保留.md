# 第 2 章 文档解析与结构保留

## 2.1 解析不是纯文本复制

标题层级决定章节边界，页码和来源行决定引用定位，代码块决定可执行实验，表格和图片决定语义。不同格式必须保留其可回溯信息：PDF 记录页码，PPTX 记录幻灯片，DOCX 记录段落/表格，Notebook 记录单元格顺序和输出。

## 2.2 Notebook 转实验说明

转换时保留：目标、环境依赖、输入数据、代码单元、关键输出、观察结论、失败提示和来源链接。不能只拼接代码文本，也不能把执行输出当成永远正确的金标；输出应标记生成时间和环境。

## 2.3 解析失败处理

扫描文件类型、大小、页数、压缩包展开体积和解析耗时。失败时记录错误码、文件哈希和可重试性，不把空文本成功写入索引。用户应看到“解析失败/部分成功/需要人工确认”三种不同结果。

## 2.4 实训检查

对同一主题准备 Markdown、PDF、PPTX 和 Notebook，比较标题、代码、表格和来源定位。验收点：任意检索块能回到文件和结构位置；解析失败能解释，不静默丢失。
