# 第 3 章 分块、元数据与索引

## 3.1 分块目标

分块要在“足够完整的语义”与“可检索的粒度”之间取平衡。章节标题、知识点、任务类型、来源位置、课程版本、owner、许可证和敏感级别应进入 metadata；正文和 metadata 的过滤职责不能混淆。

## 3.2 典型策略

- 按标题和段落优先切分，避免跨章节拼接。
- 对代码/表格使用专门边界，保留前后说明。
- 重叠只为弥补上下文断裂，不能用过大重叠掩盖结构错误。
- 同一来源重新摄入时按内容哈希和版本去重，不静默覆盖旧索引。

## 3.3 隔离

查询必须同时约束用户、组织、课程和版本范围。公开资源、系统资源和个人私有资源分开标记。删除文档要使检索和图谱同时失效，不能只删列表记录。

## 3.4 实训检查

构造两个课程和两个学生各一份同名文档。分别检索公共、课程和私有范围，确认结果不串库；修改一个 chunk 后确认旧版本仍可回放。
