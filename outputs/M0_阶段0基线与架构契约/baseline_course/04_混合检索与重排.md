# 第 4 章 混合检索与重排

## 4.1 三类检索

关键词检索擅长精确术语、代码和 ID；向量检索擅长语义相近表达；混合检索先合并候选，再通过统一分数或重排器排序。不存在对所有课程、query 和语言都最优的固定方案，必须用固定金标 query 比较。

## 4.2 query 与 top-k

记录原始 query、改写 query、过滤条件、候选数量、最终 top-k 和耗时。top-k 增大可能提高召回，也可能把无关内容塞进 context；调整参数必须绑定评测运行，不以单个成功回答判断。

## 4.3 重排注意事项

重排不能改变权限过滤，也不能凭模型偏好删除唯一证据。相关性标签应说明“支持问题”“部分相关”“无关”“冲突”，并保留排序前后结果用于分析。

## 4.4 实训检查

准备 10 个 query，分别运行关键词、向量和混合检索，输出 Recall@k、首个正确块位置和平均耗时。验收点：参数和数据版本可重现，权限过滤始终先于排序。
