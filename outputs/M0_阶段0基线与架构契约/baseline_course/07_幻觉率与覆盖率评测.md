# 第 7 章 幻觉率、适配率与覆盖率评测

## 7.1 评测先于宣称

任何指标都必须固定案例、金标、标注规则、模型/代码配置、排除项和分母。赛题给出的三项目标是评测目标；本课程只定义如何测，不预先声称当前已达标。

## 7.2 三项口径

- 幻觉率：被标注为无依据或与来源矛盾的原子事实数，除以所有可判定原子事实数；`unknown` 和无关闲聊按预先规则排除并报告。
- 画像—资源适配准确率：人工标注为符合画像难度/形式/约束的资源数，除以进入适配评测的资源数；生成失败不能从分母删除，需按案例规则计入失败或单独报告。
- 核心知识点覆盖率：获得至少一条有效讲解/实操/测量证据且通过来源/语义检查的核心知识点数，除以版本冻结的核心知识点总数。

指标报告必须同时给分子、分母、排除项、案例列表、标注者、一致性和置信区间（如适用）。

## 7.3 评测案例

案例覆盖准确性、检索、引用、画像、生成失败、审核、路径、越权、删除、断网、超时和恢复。50 个案例不能只是同一个问题重复 50 次。

## 7.4 实训检查

两名标注者独立标注同一组回答，再讨论冲突。验收点：标注依据可定位，分母不因失败而临时变化，报告可以从指标反查逐例记录。
