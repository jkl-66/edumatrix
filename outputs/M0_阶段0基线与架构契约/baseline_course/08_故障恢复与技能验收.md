# 第 8 章 故障恢复与技能验收

## 8.1 可观测闭环

用 `trace_id` 串联请求、画像快照、Agent 运行、检索候选、产物版本、质量检查、反馈事件和路径决策。日志记录错误码和摘要，不记录密钥和完整敏感上下文。

## 8.2 故障分类

- 来源/解析：格式不支持、压缩炸弹、结构丢失、超时。
- 检索：空索引、低置信、权限过滤后无结果、索引版本不一致。
- 模型：超时、限流、返回格式错误、外部服务不可用。
- 任务：重复提交、断线、部分成功、不可重试失败。
- 安全：越权、秘密泄露、危险代码、跨用户检索。

每类故障要有用户可理解的状态、可重试性和人工处理路径。

## 8.3 代表性技能验收

学习者完成“为一个 RAG 课程调整混合检索 top-k 和重排阈值，使固定 query 的证据召回改善，同时保持 owner 过滤”。系统记录参数、前后结果、引用质量、失败分支、提示依赖和隐藏测试。延迟复测用于判断保持，不以一次操作通过宣布胜任。

## 8.4 实训检查

依次模拟外部模型超时、低置信检索、删除文档后查询和断线刷新。验收点：不丢任务、不伪造成功、不泄露数据，恢复后能沿原 trace 或明确生成新 trace 并关联原因。
