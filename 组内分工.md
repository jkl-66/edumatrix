# EduMatrix 智教矩阵：核心研发团队分工与系统设计书 (全量覆盖修正版)

> [!IMPORTANT]
> **全量代码覆盖原则 (100% Codebase Coverage)**：
> 经系统性梳理，本项目包含 **14 个前端 Vue 页面/组件** 以及 **所有核心 Python 后端 API 路由**。
> 本方案已将这些模块**全量无死角地分配给 8 位成员**。确保错题本、学习笔记、对话历史、Anki 闪卡、教师看板等每一个可供展示或值得优化的点都有专人负责。
> 
> **开发边界规范**：
> 1. 本项目为**小规模运行**，无需进行高并发运维优化（不设计无锁缓存队列、分片数据库等）。
> 2. 本项目处于**受信环境**，无需隔离沙箱安全防御，普通的 Python 子进程执行即可。
> 3. 本项目**大模型 API 算力管够**，全面调用 GPT-4/星火强模型以换取最佳智能体验，不搞本地模型蒸馏。
> 4. **视觉重构要求**：全面重构系统界面，从死板的学术系统转变为流动、优美、充满高级科技感的“流光暗色模式 (Fluid Dark Mode / Glassmorphism)”，各页面交互均需加入平滑物理过渡动效。
> 
> **🤝 团队开发并行四大铁律**：
> 1. **契约先行**：开发新功能前先对齐接口。为了防止 API 接口文件冲突，API 已进行物理拆分。**必须先去 [frontend/src/api/](file:///d:/project-edumatrix/edumatrix-main/frontend/src/api/) 目录下自己模块对应的 API 子文件（如 `quiz.js`、`profile.js`）声明接口并写好 Mock 假数据格式，然后再去写后端算法**。严防后期联调车祸，实现接口层面的物理隔离。
> 2. **物理隔离**：后端路由独立，前端组件独立。**子功能模块开发者可自行在公共页面（如 `Chat.vue`）中注册挂载自己的独立组件（仅限 3 行 import 和标签代码）**，所有实际 UI 逻辑 and CSS 样式强制隔离在自己的独立组件文件中，CSS 必须加 scoped，利用 Git 智能合并消除冲突。
> 3. **绿色提交**：每次 push 前必须本地运行 `pytest test_edumatrix.py` 全绿通过。PR 支持对接人互审互并，组长离线不卡进度。
> 4. **统一基底**：本地教材、数据库脚本严格一致。初始化必须按照：`拉取教材` -> `编译向量` -> `预置学生` 三步标准命令运行。

## 👨‍💻 成员 1：学情画像与认知追踪模块 (Cognitive Profile)
**📌 模块职责与目的**：维护学生的“全息数字大脑”，精细刻画每一项技能的掌握度与遗忘时间。
**📁 负责代码与网页**：
*   **后端代码**：`profile_api.py`, `bkt_engine.py` (贝叶斯知识追踪)
*   **前端网页**：`ProfileDashboard.vue` (画像主页), `StudentAnalysis.vue` (学情深度分析页), `MasteryRadar.vue` (技能雷达组件)

> [!NOTE]
> **📊 开发强度与优化上限评估**：
> *   **开发难度**：⭐⭐⭐⭐ (偏难) — 包含 PyTorch 时序推理模型与 Poincaré 双曲流形映射，需要扎实的数学推演和前端 3D 渲染调试。
> *   **算法深度**：⭐⭐⭐⭐⭐ (极深) — 认知追踪和双曲非欧流形对齐是教育技术最前沿的学术课题。
> *   **视觉呈现**：⭐⭐⭐⭐ (高) — 动态的技能雷达与 Poincaré 3D 散点星空图需要极佳的图形学和动画微调。
> *   **优化 Ceiling (优化上限)**：**极高**。该模块是整个系统的理论根基与最性感的科技卖点，极易产出高质量的学术论文、核心专利或参赛亮点。
> *   **AI 辅助开发难度评分 (Vibecoding 警示模式)**：**8.5 / 10 (极高警告)** — 虽然 DKT 的时序推理框架和卡尔曼滤波公式 AI 能快速生成，但**双曲流形空间（Poincaré Disk）的非欧几何梯度求导与坐标对齐是 AI 的重度幻觉区**。如果负责队员自己完全不理解原理，盲目复制 AI 代码，在面对维度不匹配或非欧距离计算发散时，会陷入无限循环报错。

### 任务 1：基于时序数据的轻量级深度知识追踪 (DKT) 引擎
*   **具体实现细节**：在 `profile_api.py` 中建立一个轻量级 PyTorch 时序推理模型，加载预训练的 DKT 权重。每次触发学情刷新时，从 `quiz_records` 提取最近 50 次交互序列，输入模型，将输出更新至 `student_profiles` 的 `cognitive_map` 字段。
*   **大白话解释**：把学生过去做题的历史记录像看电影一样放给算法看，预测他明天做新题能不能做对。
*   **研发目的与价值**：能在学生真正忘记知识点之前，提前预测弱项，给复习系统提供最精准的数据源。

### 任务 2：卡尔曼滤波防抖与状态平滑估计器
*   **具体实现细节**：在 `profile_api.py` 中编写卡尔曼滤波数学类，设定系统噪声协方差 $Q$ 和测量噪声协方差 $R$。增量更新画像时，通过“预测-更新”两步迭代，输出平滑后的掌握度分值写入数据库。
*   **大白话解释**：防止学生偶然考砸一次，系统就误判定他是学渣，能识别出“他这次只是粗心”。
*   **研发目的与价值**：保证学生成长轨迹的平滑和客观性，避免因为一两次手滑导致系统误推荐简单题。

### 任务 3：基于双曲几何 (Poincaré Disk) 的认知流形对齐算法与 3D 可视化
*   **具体实现细节**：在 `manifold_alignment.py` 中，使用 Poincaré 距离公式替代欧式距离，进行学生掌握向量与目标大纲向量的仿射映射。前端引入 `ManifoldVisualizer.vue`，用发光的双曲圆盘投影图可视化学生认知状态的逼近过程。
*   **大白话解释**：把杂乱的知识点放在一个发光的 3D 双曲圆盘里，看看学生离通关终点还有多远。
*   **研发目的与价值**：用先进的几何表征直观展示复杂的跨学科掌握层次，让学生对自己的学力有上帝视角的感知。

🛠️ **自行拓展方向**：
1. **错题自动归类**：自动将做错的题目贴上“粗心”、“公式记错”或“概念混淆”标签。
2. **组队匹配系统**：对比两个学生的画像，找出擅长科目互补的两个人推荐他们组队。
3. **成长里程碑勋章**：学生点亮了某个高难度知识树分支时颁发虚拟勋章。
4. **学科天赋测试器**：通过答题速度和反应，估算学生更偏向理科还是文科。
5. **遗忘红点预警**：在软件主页上，如果某个重要公式快忘了，像微信红点一样提示。

> [!TIP]
> **💡 自由探索指南**：以上拓展方向仅供参考。学情追踪的本质是数据洞察与诊断，我们非常鼓励你打破常规，自由探索任何你认为更好玩、更利于大盘演示或更有学术创新价值的学情可视化和画像深度挖掘方案，开发完全不用拘泥于本任务书！

## 🗺️ 成员 2：自适应路径规划与复习模块 (Path Planning & Revision)
**📌 模块职责与目的**：像导航仪一样，为学生规划最优的学习复习路线，提供打卡 and 闪卡学习。
**📁 负责代码与网页**：
*   **后端代码**：`learning_strategy.py`, `flashcard_api.py` (闪卡API), `anki_engine.py` (Anki记忆引擎), `review_plans` 表
*   **前端网页**：`LearningPathGraph.vue` (路径图谱页), `RevisionCalendar.vue` (复习日历打卡页), `Review.vue` (复习卡片操作页)

> [!NOTE]
> **📊 开发强度与优化上限评估**：
> *   **开发难度**：⭐⭐⭐ (中等) — 主要涉及经典图算法与状态机调度，数据流与逻辑链路清晰，开发阻力适中。
> *   **算法深度**：⭐⭐⭐⭐ (高) — 跨学科的启发式图搜索与基于 SM-2 的复习排程逻辑，需要扎实的离散数学与图论算法根基。
> *   **视觉呈现**：⭐⭐⭐ (中等) — 复习日历和卡片栈在前端主要考验组件库的优雅交互和动效。
> *   **优化 Ceiling (优化上限)**：**高**。这是让系统跑起来、产生真正学习效果的“引擎核心”，学习路径生成机制和卡片精细调优的策略上限很深。
> *   **AI 辅助开发难度评分 (Vibecoding 模式)**：**4.5 / 10 (低难度)** — A* 寻路和 SuperMemo-2 复习算法是高度标准化的学术算法，AI 在网络拥有海量的成熟代码库。只要输入明确的参数结构，AI 可以在 10 秒内吐出完美逻辑，人机联调只需验证路径的逻辑合理性。

### 任务 1：跨学科微概念 (Micro-concept) 语义关联图谱构建
*   **具体实现细节**：使用图嵌入 (Graph Embedding) 算法（如 Node2Vec）计算不同学科概念在极小空间的空间相似度。在 `learning_strategy.py` 中使用 NetworkX 维护一个多学科混合大图，寻找物理与数学前置节点。
*   **大白话解释**：在后台把“物理的电磁学”和“数学的积分”用线连起来，打通学科壁垒。
*   **研发目的与价值**：当学生物理学不懂时，AI 能够发现他其实是数学没学好，带他去隔壁复习数学。

### 任务 2：基于前置约束的自适应拓扑前置展开与 A* 跨域类比混合寻路机制 (Hybrid Routing)
*   **具体实现细节**：在单一学科体系内，使用拓扑前置展开算法逆向提取未掌握节点并排序，以防跳过前置物理防线；同时，激活并使用 A* 启发式寻路算法在跨学科语义网络（含 GraphRAG 关系与 Embedding 相似度）中，寻找从已掌握跨学科概念到目标概念的最短代价值辅助类比支路路径，提供“脚手架”支撑推荐。
*   **大白话解释**：在学科内部严密排程，绝不漏掉基础课；在遇到障碍时，用 A* 算法帮你从其他学科（如数学底子）里规划一条最近的类比解释路径，搭桥梁帮你跨界学习。
*   **研发目的与价值**：攻克了单纯 A* 寻路极易漏掉严格前置依赖的教育学漏洞，实现图谱拓扑合规防抖与跨学科语义类比的完美融合。

### 任务 3：个性化学情复习排程引擎 (Anki SM-2)
*   **具体实现细节**：在 `review_plans` 表中存储记忆因子、最近复习时间、推送间隔天数。每次测试反馈后，动态调节其复习卡片对应的记忆因子 (E-Factor) 和下次推送时间。
*   **大白话解释**：自动记下你每个闪卡的难易程度，在你要忘掉的那天提醒你拿出来在网页上打卡复习。
*   **研发目的与价值**：提供高度个性化的复习计划，用最少的复习次数达成最持久的记忆效果。

🛠️ **自行拓展方向**：
1. **期末通关倒计时**：设定考试时间，自动计算并排程每天每天必须要看多少卡片。
2. **10分钟微学习**：输入碎片时间，系统自动切出一段 10 分钟能看完的微型闪卡。
3. **跨学科趣味彩蛋**：在路径中，随机安插一些和概念相关的科学家历史八卦。

5. **降级科普备用路**：如果一个概念卡关了三天，系统自动换路成漫画科普。

> **💡 自由探索指南**：以上拓展方向仅供参考。路径推荐和卡片复习是自适应教育“提升效率”的关键，如果你想到了更奇妙的路径图谱推荐模型、或者更有吸引力的游戏化卡片打卡玩法，完全可以跳出已有框架，尽情施展脚力！

## 📚 成员 3：多模态图谱联邦与知识库模块 (Knowledge Base & RAG)
**📌 模块职责与目的**：负责教学教案的图文吞吐、图片公式检索，提供外援事实。
**📁 负责代码与网页**：
*   **后端代码**：`rag_engine.py`, `document_parser.py`, `vector_store_faiss.py`, `knowledge_api.py`, `web_search_api.py`
*   **前端网页**：`Knowledge.vue` (教案上传与检索管理页)

> [!NOTE]
> **📊 开发强度与优化上限评估**：
> *   **开发难度**：⭐⭐⭐ (中等) — 重点在于多模态 API 调用、增量文件对比以及 FAISS 数据对齐，工程量大过纯算法难度。
> *   **算法深度**：⭐⭐⭐ (中等) — 核心在于构建分块 (Chunking) 的启发式规则与图谱合并规则，主要是调优工作。
> *   **视觉呈现**：⭐⭐ (易) — 标准的前端文件上传列表、多模态预览，页面逻辑属于常规交互。
> *   **优化 Ceiling (优化上限)**：**中偏高**。由于 API 费用充足，直接利用视觉大模型处理文档即可达到天花板，后期的优化上限主要在增量图合并的拓扑正确性上。
> *   **AI 辅助开发难度评分 (Vibecoding 模式)**：**5.0 / 10 (中等难度)** — PDF图片渲染、Qwen-VL 的 API 发包接收、FAISS 本地库加载都是常规系统工程。AI 能一键处理好文件读写和检索对齐，主要难度在于调试大模型增量提取图谱时的 Prompt 稳定性。

### 任务 1：基于开源多模态大模型的视觉文档 RAG 管道
*   **具体实现细节**：在 `document_parser.py` 中使用 PyMuPDF 将教案 PDF 的每页渲染为高清图片。直接通过 HTTP 接口调用多模态大模型（如 Qwen-VL 或 GPT-4o-mini）提取图片的稠密表征，直接与用户的提问进行向量匹配。
*   **大白话解释**：AI 找课外资料时不仅看文字，还能直接看懂课件里的插图和手写公式。
*   **研发目的与价值**：彻底攻克理科教学中“图片、公式无法检索”的痛点，实现真正的图文多模态检索。

### 任务 2：增量式局部图谱更新与自生长 GraphRAG
*   **具体实现细节**：在 `ingestion.py` 中对新上传文章进行句级别 diff。对差异块调用大模型提取关联关系，通过图合并算法热更新 FAISS 及 SQLite 中的图谱关联。
*   **大白话解释**：每次老师传一份新教案，系统自动把新的知识点织进已有的知识网里，不用大动干戈。
*   **研发目的与价值**：提供结构化的背景知识关联，回答时不仅吐出文字，还能画出实体关系图。

### 任务 3：跨模态特征潜空间对齐实验 (Text-Image-Formula)
*   **具体实现细节**：在收集公式-图解对齐数据集，利用对比损失函数微调一个特征映射层，使得公式向量、图片向量和文字向量的余弦距离拉近。
*   **大白话解释**：让系统能够实现“用文字搜公式”、“用公式找图片”的跨界搜索。
*   **研发目的与价值**：实现真正意义上的多模态混合搜索，提升 RAG 检索在理科场景下的精确度。

🛠️ **自行拓展方向**：
1. **草稿纸自动美化**：学生拍一张乱七八糟的手写草稿图，AI 自动排版成打印版 LaTeX。
2. **视频课件自动打点**：如果教材里有视频，自动在进度条上标记出“这分钟讲了什么”。
3. **学术论文降维翻译**：检索出很难的高校论文，系统自动“降维”翻译成初中生故事。
4. **图表悬停交互解说**：课件图画里的线条，鼠标悬停时会显示气泡解释它是干嘛的。
5. **私人文档混搜**：允许上传自己学校的试卷，和系统教材混在一起搜索。

## ⚔️ 成员 5：逻辑辩论与对抗互动模块 (Debate & Socratic)
**📌 模块职责与目的**：通过“杠精”式的学术 PK，强迫学生动脑，杜绝 AI 自身幻觉。

> *   **开发难度**：⭐⭐⭐⭐ (偏难) — 难点在于精准训练大模型输出符合 SymPy 解析规范的公式，且需要调试多 Agent 对局循环（Self-play）。
> *   **算法深度**：⭐⭐⭐⭐ (高) — 融合了神经符号 AI（大模型 + 符号代数求解器）和简单的博弈学习算法，学术性极强。
> *   **视觉呈现**：⭐⭐⭐ (中等) — PK 卡片、倒计时状态机以及结算音效，页面动效较多。
> *   **优化 Ceiling (优化上限)**：**极高**。作为系统最富趣味性和思辨性的创新点，算法优化（如出题难度曲线）的上限空间几乎无限，非常容易做成比赛和演示中的核心高亮环节。
> *   **AI 辅助开发难度评分 (Vibecoding 警示模式)**：**8.0 / 10 (高难度警告)** — 神经符号系统（大模型+Z3/SymPy）具有极高的格式敏感性。大模型极易在复杂的数学推导中输出语法错误的符号。如果队员完全不理解原理，盲目复制 AI 代码，在面对 SymPy 解析器频繁崩溃时，会陷入 AI 解释不清的报错泥潭，必须人工监督 AI 的 Prompt 模板格式。

### 任务 1：外挂符号数学引擎 (SymPy) 的防幻觉辩手
*   **具体实现细节**：编写 Prompt 要求 Debater 输出固定格式 of SymPy 符号代数。后端拦截器将这段符号代码提取并送入 Python 子进程沙箱运行，验证等式两边是否成立，成立方可吐流。
*   **大白话解释**：AI 在跟学生辩论理科题前，自己先用计算器验算一下思路对不对，免得自己防卫过当胡说八道。
*   **研发目的与价值**：彻底攻克 LLM 无法进行严密数理逻辑推理的先天死穴，确保对话中的学术结论 100% 正确。

### 任务 2：难度动态自适应的自我博弈 (Self-Play) 对抗出题器
*   **具体实现细节**：后端启动一个对局循环：出题 Agent 埋入 Bug，解答 Agent 尝试解题。统计解答 Agent 发现 Bug 的成功率。通过调整 Prompt 中的“漏洞隐蔽系数”输出完美适配的题目。
*   **大白话解释**：让两个 AI 晚上在后台自己下棋玩，找出最能难住你、但你又能做出来的题目。
*   **研发目的与价值**：动态生成完美匹配学生能力“最近发展区 (ZPD)”的高质量对位题，让做题不再痛苦。

### 任务 3：基于对话熵 (Dialogue Entropy) 监测的辩论控制
*   **具体实现细节**：每次对话计算最新几轮的 Sentence Embedding 距离。当距离波动极小、熵值居高不下时，调度官 `Coordinator` 触发拦截，发出“总结语”并切出辩论状态。
*   **大白话解释**：当 AI 发现你和它在原地绕圈子抬杠时，它会主动停下来，拍拍你说“好啦，我们看标准答案吧”。
*   **研发目的与价值**：节省大模型的无效算力消耗，防止学生因过度辩论导致挫败感，掌控课堂的节奏。

🛠️ **自行拓展方向**：

*   **大白话解释**：不要做 50 道死卷子，而是做对一题就变难，做错就变简单，10道题测出真实等级。

*   **大白话解释**：用电脑模拟几万个学霸 and 学渣来我们系统做题，倒推算出一道题到底是难是易。

> *   **AI 辅助开发难度评分 (Vibecoding 警示模式)**：**9.5 / 10 (地狱难度警告)** — **AI 无法染指的死角**。WGSL 着色器开发在整个 AI 知识库中样本极度匮乏，且大模型完全无法运行调试显卡渲染器。显卡报错通常只返回毫无意义的显存错误代码。盲目复制报错给 AI 会导致渲染逻辑彻底崩溃，开发人员必须具备极强的 WebGPU/WebGL 图形学功底。

### 任务 1：增量式虚拟 DOM (VDOM) 修补的流式 Markdown/公式排版渲染机
*   **具体实现细节**：在前端拦截 SSE 管道推送的字符流。运行自研的微型增量解析状态机，识别出完整的 `$$...$$` 或 ````mermaid...```` 闭合块，再调用本地的 KaTeX/Mermaid 渲染，其余未完成的部分渲染为骨架屏占位。
*   **大白话解释**：当大模型打字吐公式时，网页稳如泰山，公式是一排排滑出来，而不是一闪一闪地跳。
*   **研发目的与价值**：提供无懈可击、极具高级感和流畅度的主聊天区交互，解决市面上多数大模型系统公式重绘跳闪的通病。

### 任务 2：基于长上下文对话的结构化讲义一键生成与 PDF/LaTeX 导出
*   **具体实现细节**：设计 LaTeX 模板，定义公式、图表的展示样式。后端接收指令后，将清洗出的结构化笔记填充到模板中，通过子进程调用 Pandoc/Weasyprint 编译，生成 PDF 提供下载。
*   **大白话解释**：把之前的长篇聊天文本讲义线索去掉，自动提炼总结排版，一键导出一份排版非常漂亮的 PDF 复习讲义。
*   **研发目的与价值**：让学习过程有实体资产的沉淀，方便学生下线打印、实体复习，提升产品的实用属性。

### 任务 3：几十万节点力导向图的 WebGPU 着色器下放加速
*   **具体实现细节**：在前端使用 Three.js 的 WebGPURenderer 或者编写原生 WGSL 着色器。将包含学科节点坐标 and 连线的矩阵传入显存，用 GPU 完成力导向计算和发光 Bloom 后处理。
*   **大白话解释**：把原本会让网页卡死的海量连线图，利用显卡加速，变成 3D 电影里那种可以顺滑拖拽的星空效果。
*   **研发目的与价值**：突破前端浏览器性能天花板，用极致的技术实力做出让客户和投资人“看一眼就 Wow”的黑科技数据呈现。

### 🎨 界面重构与视觉美化任务：全局流光设计系统 (Theme Tokens) 与发光 3D 图谱微动效
*   **具体实现方案**：
    1. 制定全局流光设计变量（CSS Theme Tokens），在 `index.css` 中配置暗夜紫、极光蓝、微光灰等渐变色彩，以及全局卡片的阴影呼吸动效。
    2. 在 3D 星图的 Shader 中，为悬浮选中的概念节点编写后处理发光 Bloom 着色器。当鼠标指上去时，该节点向周围扩散环形光波（类似物理波动），展现出学科概念的引力场。
*   **大白话解释**：定一套全站统一的“高端黑客帝国风”流光背景和透明玻璃卡片规范，并且让 3D 星图的球球点一下会像水波一样发光。
*   **研发目的与价值**：给整个系统定下最核心的高级主基调，将视觉冲击力拔到最高，形成产品的第一视觉护城河。

## 📊 成员 8：大盘数据管理与系统配置模块 (Dashboard & Admin)

*   **前端网页**：`Dashboard.vue` (学生概览舱), `Teacher.vue` (教师管理面板), `Settings.vue` (系统设置页), `Login.vue` (登录注册页)

### 任务 1：多维隐式特征“认知负荷”估计器与行为收集
*   **具体实现细节**：在 `Chat.vue` 的输入框上监听 `@keyup`，记录打字间隔微秒数存入数组。发回后端后，用一笔简单的加权乘法微调路由器的切换概率。结合 `behavior_api.py` 统计做题错率与修改频次。
*   **大白话解释**：根据你打字犹豫的时间、删删改改的次数，悄悄且极速地推算出你现在是不是学累了。

*   **大白话解释**：AI 能根据你的精力水平自动场控，状态好就带你打硬仗，疲惫了就主动跟你开个玩笑放松。
*   **研发目的与价值**：让整个教学流更加符合人类学习的内在节律，提高学生学习效率。

### 任务 3：游戏化情境战役与沉浸式互动教学生成器

*   **大白话解释**：把枯燥的物理公式变成文字冒险生存游戏，做对选择题就能逃生，玩着玩着就学会了。
