## 🧑‍💻 Role A（队长/后端）：高并发基建、数据安全与容器部署

### 时间分配与详细实施步骤

#### 【阶段一：工程初始化与安全多租户重构】 (第 1 - 4 天)
* 任务 1.1：初始化工程与治理锁定。初始化 Git 仓库，锁定 Python 3.11+。配置 `.pre-commit-config.yaml`，全量集成 ruff、isort 和 mypy，强制每次 commit 触发代码规整。
* 任务 1.2：环境变量与配置加载。引入 `python-dotenv` 建立 `.env`，物理隔离 API 密钥和数据库连接。用 Pydantic BaseSettings 写 `EduMatrixConfig`，严禁代码硬编码敏感数据。
* 任务 1.3：独立 Schema 动态数据源隔离。废弃单表字段过滤，用 `contextvars` 声明 `tenant_context`（默认 public）。利用 SQLAlchemy 的 `before_cursor_execute` 拦截事件，在 SQL 发出前强行注入 `SET search_path TO {current_tenant};`，杜绝并发串线。
* 任务 1.4：连接入池物理洗白（重点加固）。利用 SQLAlchemy 连接池的 `checkin` 事件，当连接放回池中瞬间，强行异步执行 `SET search_path TO public;`，防止公共任务复用脏连接。

#### 【阶段二：异步网关、限流与防空耗机制】 (第 5 - 10 天)
* 任务 2.1：跨域与日志拦截。封杀 `allow_origins=["*"]`，精准配置为前端白名单（如 localhost:5173）。中间件里加 `time.perf_counter()`，把耗时和 Trace ID 写入标准化日志。
* 任务 2.2：鉴权与云端熔断。接入 JWT 鉴权。在星火 API 方法上套 `pybreaker` 异常熔断器（fail_max=3, reset_timeout=30.0）。超时 3 次自动切回调取本地 vLLM 驱动的备用节点。
* 任务 2.3：本地算力令牌桶保护。在本地网关加 `asyncio.Semaphore(5)`，限制本地核心并发上限为 5，满了自动进入后端非阻塞排队，防爆显存。
* 任务 2.4：协程孤儿自杀拦截。在 `/api/chat/stream` 内部循环里加 `if await request.is_disconnected(): break`。前端一断开，后端秒抛 cancel() 强杀协程。
* 任务 2.5：多轨通讯。打通流式推流路由，并开辟 `@app.websocket("/ws/status")` 通道，向前端推智能体运行的底层状态码胶囊。

#### 【阶段三：垃圾回收与代码沙盒常驻】 (第 11 - 15 天)
* 任务 1.5：本地自适应垃圾回收。挂载 FastAPI 的 `BackgroundTasks`，每次新建 Session 触发异步 GC，删除 created_at 超 7 天且未打留存标签的历史数据（白名单豁免 `demo-student`）。
* 任务 6.3：隔离代码沙盒常驻池。废弃临时拉起容器。后端启动时，预先拉起 3 个断网、纯净的 Python 容器常驻，配 `mem_limit="512m"` 和 `nano_cpus` 限制单核。生成代码后用 `container.exec_run` 跑，后台加 2.0 秒轮询看门狗，超时直接 `container.kill()` 强熔断。

#### 【阶段四：后端部署与压测交付】 (第 35 - 40 天)
* 任务 10.2：系统一键容器化打包。写多阶段构建 Dockerfile，把 Vue3 前端和 Python 后端合并进单一镜像，减小体积。写 `start.sh` 集成 Alembic 初始化和常量 seeding。
* 压测配合：写 `tests/locustfile.py`，配置 HttpUser 轰炸流式网关，生成 50 并发下系统 0 崩溃的压测图表，塞进设计说明书。

## 💻 Role B（算法）：RAG 引擎、Swarm 对齐调度与学术微调

### 时间分配与详细实施步骤

#### 【阶段一：教材解析与多模态双轨切片】 (第 1 - 7 天)
* 任务 3.1：真实语料切块。收集权威 PDF 教材，用 `RecursiveCharacterTextSplitter` 切块，大小 500 字符，重叠 75 字符（重叠率 15%）。
* 任务 4.1：公式版面解析与还原。放弃纯 CLIP。用 PaddleOCR 或 LayoutLMv3 做版面定位划分出公式区。调用 `LaTeX-OCR` (Pix2Text) 把公式 100% 还原为标准 LaTeX 字符串。
* 任务 4.2：双轨增强嵌入。在丢给 `BAAI/bge-m3` 前，用 Jinja2 把公式变成 `[LaTeX_Source: ...] (公式语义解释: ...)` 拼装结构，解决分词器稀疏化断层。

#### 【阶段二：Neo4j 动态图谱与混合检索】 (第 8 - 14 天)
* 任务 3.2：三元组自动化抽取。写 `graph_builder.py`，用本地 Qwen 引擎强制输出提取依赖关系的 JSON 数组。通过 Cypher 的 `MERGE` 语句异步写入 Neo4j。
* 任务 3.3：实体白名单对齐。写入图谱前，先比对系统白名单；未命中则算 Levenshtein 编辑距离（<=2强行归一）；还未命中则算 Embedding 相似度（>0.85归一），消除同义词噪音。
* 任务 4.3：稠密/稀疏混合与提词器。挂载术语重写层（例如碰到“拉氏算子”补全近义词）。把 BM25 和向量得分融合，召回候选后再用 `bge-reranker-large` 重排打分。

#### 【阶段三：Swarm 异步调度与坍塌自愈网】 (第 15 - 25 天)
* 任务 5.1：矩阵全异步解耦。在 `agent_swarm.py` 里干掉 ThreadPoolExecutor，用 `asyncio.gather()` 并发调 5 个生成 Agent，外裹 `asyncio.wait_for(timeout=15.0)`。
* 任务 5.2：Prompt 延迟渲染合并。把 10 维画像从大段文字精简成一行 JSON 串（如 `{"states":[4,3,5...]}`），放进 User Prompt 末尾，压低首字延迟到 0.6 秒。
* 任务 6.1：对齐 advice 参数化。当双曲流形对齐发现符号冲突，直接输出强约束参数 JSON 补丁（如 `{"target_api": "nn.MaxPool2d"}`）。
* 任务 6.2：Guided Decoding 概率坍塌自愈防线（重点加固）。在代码精炼 `coder.py` 中，包裹 `try...except ValidationError` 捕获。若 vLLM 结合 Schema 时发生空流坍塌，0 毫秒原地短路，用 `re.sub` 规则引擎强行替换（如把 AvgPool 物理替换成 MaxPool），保住 3 秒出流。

#### 【阶段四：沙盒提炼与 A800 本地微调】 (第 26 - 40 天)
* 任务 11.1：112 道难题自动化测试。把偏微分/矩阵难题转成 `JSONL`，写流水线脚本批量发问、正则提答案，加 Checkpoint 防止断网。
* 任务 13.1：并网符号智能体。在 `models.py` 加入 `VerifierAgent`，调本地 `sympy` 库自动验算极客助教的求导积分公式，错就报 `AstVerificationError` 触发回滚。
* 任务 12.1 & 13.2：沙盒账本提炼与 LoRA 微调。从前期压测沙盒提取 2000 条 Alpaca 格式错因反思数据。在 A800 上用 LLaMA-Factory + `Qwen2.5-Math-7B-Instruct` 做 LoRA 微调（`bf16=True, lr=2e-5`）。跑出 Base vs DRAG vs Refiner-Verifier 的消融数据写进顶会论文。

## 🎨 Role C（前端）：流式多模态渲染、缓冲拦截与视觉包装

### 时间分配与详细实施步骤

#### 【阶段一：基座搭建与 SSE 缓冲看门狗】 (第 1 - 10 天)
* 任务 7.1：前端底座。用 Vite 初始化 Vue3 + TS，配好 TailwindCSS 样式，挂载 Pinia 状态中心。
* 任务 7.2：语法缓冲拦截器 (Buffer Filter)。修改 `src/stores/chat.ts`，检测到 `$$` 或 ````mermaid` 时，状态机切入 `BUFFER` 模式，拦截碎片不给 DOM 渲染，解决公式/代码没闭合就渲染导致的控制台爆红。
* 任务 7.2 & 7.3：自动补全降级看门狗。配置 700 字符计数器和 `[DONE]` 检测。如果 Buffer 锁定时超长未闭合，物理熔断，用脚本给末尾强行补上 `$$` 或 ````\n` 再去渲染，保网页不崩。
* 任务 8.1：UI 柔和补偿。看门狗触发补全时，激活 `is_truncated = true`，自动在破损版面下贴个淡黄色提示：“前端防御网已为您拦截并自动修复版面”，把硬伤包装成产品特性。

#### 【阶段二：时间轴、雷达图残影与多模态组件】 (第 11 - 20 天)
* 任务 8.2：动态协作时间轴。右上角搓个横向组件，听后端 WS 发来的状态码，用 `animate-pulse` 依次点亮：苏格拉底接单 -> 探针画像 -> DRAG 开庭 -> 对齐质检。
* 任务 8.3：双圈对照雷达图。用 `vue-echarts`，重构 `series`，同时画淡灰色底圈（初始学情）和科技蓝顶圈（当前学情），展示自适应教学扩圈的政绩成效。
* UI 强提示 (任务 9.3)：显著位置挂星火大模型 logo。Tabs 固定“讲义、导图、沙盒、练习、脚本”五大常驻区，贴“已通过双曲流形质检”绿色 Badge。

#### 【阶段三：TTS 流式发声与嘴形滤波】 (第 21 - 30 天)
* 任务 9.1：流式 TTS 替换。彻底弃用死等几分钟的长视频 API，对接讯飞 TTS WebSocket，用 Web Audio API 接 base64 流实现毫秒级零排队发声。
* 任务 9.2：一阶低通滤波嘴形联动（重点加固）。加装 Lottie 二维头像。通过 `createAnalyser()` 抓振幅。写一个类套低通指数平滑算法，锁死系数 `alpha = 0.25`，让缩放 75% 靠上一帧物理惯性，25% 吸纳新振幅，彻底抹平爆破音抽搐。

#### 【阶段四：离线导出 PDF 与验收配合】 (第 31 - 40 天)
* 任务 10.1：一键导出离线错题本。在 `src/styles/print.css` 写 `@media print` 媒体查询，用 `print:hidden` 类把聊天框、仪表盘隐藏，只留讲义公式题库。绑按钮调原生 `window.print()`。
* 全盘验收配合：配合跑通 50 并发压测。录制王牌演示视频，给双圈雷达图残影、数字人平滑嘴形特写。
